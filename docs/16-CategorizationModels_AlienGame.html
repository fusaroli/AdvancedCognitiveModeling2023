17.1 Introduction

This chapter simulates the three categorization models (Exemplar/GCM, Prototype/Kalman Filter, Rule-Based/Particle Filter) on a task adapted from Tylén et al. (2023). In this task, participants (or models) learn to categorize visually complex “alien” stimuli based on feedback.

17.1.1 Stimuli and Features

The aliens vary along five binary features:

  • f1 — Legs: Slim (0) / Fat (1)
  • f2 — Arms: Downwards (0) / Upwards (1)
  • f3 — Body Dots: Absent (0) / Present (1)
  • f4 — Eyes: Normal (0) / On Stalks (1)
  • f5 — Color: Green (0) / Blue (1)

Each alien stimulus is a 5-dimensional binary vector (e.g., c(0, 1, 0, 1, 1) means slim legs, arms up, no dots, eyes on stalks, blue color). There are \(2^5 = 32\) unique aliens.

17.1.2 Categorization Dimensions and Task

Participants must react to each alien on two independent binary categorizations:

  • Dangerous: Is the alien dangerous (1) or peaceful (0)?
  • Nutritious: Is the alien nutritious (1) or non-nutritious (0)?

This results in four possible response combinations. The goal is to learn the underlying rules or patterns that determine each dimension based on the alien’s features.

17.1.3 Experimental Sessions and Complexity

The task involves three sessions with increasing rule complexity. Each session consists of three blocks where the 32 unique aliens are presented once per block in random order, with feedback provided after each choice.

N.B. the article also discusses a test session where participants are presented with new aliens. This is not implemented here since the novelty is in how the features are visually represented, and that change is transparent to our models.

17.1.4 Category Rules per Session

Based on Tylén et al. (2023):

Session 1 (Low Complexity):

  • Dangerous = f1 (Fat Legs) == 1 AND f3 (Dots) == 1
  • Nutritious = f4 (Eyes on Stalks) == 1

Session 2 (Intermediate Complexity):

  • Dangerous = f4 (Eyes on Stalks) == 1
  • Nutritious = At least 2 out of f1, f2, f3 are 1 (i.e., f1 + f2 + f3 ≥ 2)

Session 3 (High Complexity):

  • Dangerous = f4 (Eyes on Stalks) == 1 AND f5 (Blue Color) == 1
  • Nutritious = (f2 + f3 == 2) OR (f1 == 1 AND (f4 + f5 ≥ 1) AND (f2 + f3 ≥ 1))

17.1.5 Modeling Goal

We will simulate how the three categorization models learn the ‘Dangerous’ and ‘Nutritious’ categories across these three sessions. We are interested in comparing their learning trajectories (accuracy over trials) under different parameter assumptions — attention weights for the GCM, noise level for the Kalman filter, and rule complexity for the Particle Filter.

17.2 Phase 0 for the Alien Game

Chapters 12–14 each opened with a Phase 0 interrogation that surfaced the structural commitments buried in the model code. This chapter inherits all of those commitments — and then adds three more, all stemming from the move to a new task. We surface them before any code runs, because the cleanest place to catch a hidden assumption is before it propagates into 800 lines of simulation.

Q1 — What changes when we move from 2D continuous features to 5D binary features?

Three concrete things, one per model family:

  • For the GCM: attention weights are now 5-vectors on the simplex rather than 2-vectors. The Manhattan distance \(d(x, x_i) = \sum_m w_m |x_m - x_{i,m}|\) reduces (because all features are binary) to a weighted Hamming distance — the sum of weights on the dimensions where the two stimuli differ. This is a strict simplification of Chapter 12’s continuous-feature setting, and it makes the GCM’s decision rule especially transparent: similarity is “agreement on the features the model attends to.”

  • For the Kalman prototype model: features now live in \(\{0, 1\}\), so the prototypes live in the unit cube \([0, 1]^5\). Chapter 13 used initial_sigma_diag = 10.0 for a 2D continuous space where features ranged over \([1, 4]\) — that prior covariance was already wide but defensible. Carrying 10.0 into the 5D binary setting would put the prior covariance at roughly 100× the feature variance, which is an absurdly diffuse prior on the unit cube. We commit here to initial_sigma_diag = 1.0, which puts the prior covariance on the same order of magnitude as the feature variance and matches Chapter 17’s value when the same model is fitted to human data.

  • For the rule particle filter: thresholds are no longer drawn from a continuous range. The only meaningful threshold for a binary feature is 0.5, the midpoint between 0 and 1. The operators > and < then collapse to “feature equals 1” and “feature equals 0,” respectively. The rule grammar from Chapter 14 — \(r = (D, T, O, L, c)\) — survives this transition unchanged in form, but the threshold space is now a single point per dimension, making the grammar finite and discrete rather than combinatorial-times-continuous. This is easier for the cognitive theory to handle, not harder.

What we commit to here: the dimensionality change is genuine and is named explicitly in the code; the Kalman initial covariance is fixed at 1.0; the rule grammar’s threshold space collapses to {0.5} per dimension.

Q2 — Why fit each decision dimension separately, rather than jointly?

The Alien Game asks two binary questions per trial — “Dangerous?” and “Nutritious?” — and earlier drafts of this chapter, like the implementation in this chapter, fitted each dimension independently. There are at least three alternatives:

  1. Joint 4-way categorical model. A single 4-class outcome \(\{(\text{D}, \text{N}), (\text{D}, \neg\text{N}), (\neg\text{D}, \text{N}), (\neg\text{D}, \neg\text{N})\}\), fitted with a softmax. This couples the two decisions and lets the model represent dependencies between them.
  2. Shared-attention joint model. Two binary outcomes, but with a shared attention weight vector across the two decisions (the agent is constrained to attend to the same features for both).
  3. Independent fits (what we do). Two binary outcomes fitted with separate parameters and separate sequential states.

What we commit to here: independent fits. The cognitive claim is that the agent learns the two dimensions independently, with no transfer of attention or hypothesis content between them. This is defensible because the experimental design treats the two dimensions as separate questions with separate feedback channels, but it is a non-trivial claim — a real learner might allocate attention jointly, or notice a feature is informative for one dimension and try it for the other. We accept this commitment for pedagogical economy; relaxing it is a natural extension.

Q3 — Has the data been verified against the assumed rules?

Earlier drafts of this chapter contained a “bug fix note” tucked between two code blocks, mentioning that the Session 1 Dangerous rule had been corrected from f2 & f3 to f1 & f3 (Fat Legs, not Arms Up), and that a comment typo in the Session 2 Nutritious description had been fixed. These corrections matter — the GCM’s optimal attention weights below depend on knowing the true rule. The honest place for this information is in a Phase 0 verification block that runs once and halts the chapter if any rule fails to match its expected definition.

Below in §“Setup: Stimuli and True Category Labels,” we generate the category labels under each session’s rule and then verify the labels match what the rules should produce. The check is built into the data construction. If a future revision of the chapter changes the rule definitions or the feature mapping, the verification block will catch the inconsistency before it propagates.

Q4 — The rule particle filter in this chapter is Model A, not Model B.

Chapter 14 made a load-bearing distinction between two models with the same name:

  • Model A = the particle filter — the cognitive theory. Sequential, path-dependent, with a combinatorial rule space. Likelihood intractable in HMC. Fitted via Approximate Bayesian Computation (ABC).
  • Model B = a static marginalised mixture over a fixed candidate rule set, written in Stan. Closed-form likelihood. Conditionally exchangeable. A biased proxy for Model A.

The rule simulator in this chapter is Model A. It is the cognitive theory itself, run forward to generate predictions. Model B does not appear in this chapter at all, because Model B has no learning curve and the chapter’s purpose is to compare learning curves. When Chapter 17 fits the rule architecture to human data, it will fit both models — Model B via HMC for the proxy comparison, and Model A via ABC for the cognitive comparison — and report the gap between their posteriors as the structural-bias signature, exactly as Chapter 14 does on simulated data. This chapter’s job is to set up the cognitive predictions Model A produces, so that the ABC-fitted Model A in Chapter 17 has something to be evaluated against.

What we commit to here: the particle filter we run in this chapter is the cognitive theory. We do not run Model B. Comparison between the two happens in Chapter 17 on real data via ABC.

Q5 — Are we relaxing any of Chapter 14’s structural commitments?

Yes. Chapter 14 committed to:

  • N_PARTICLES_FIXED = 100 (a working-memory bound on hypothesis count)
  • MAX_RULE_DIMS = 2 (rules use at most two feature conditions)
  • Operators {>, <} and logical connectives {AND, OR}

This chapter intentionally varies N_particles ∈ {50, 150, 250} and max_dims ∈ {1, 2, 3} in the simulation grid below. Both relaxations are deliberate and have cognitive content:

  • N_particles is varied because we want to see how learning speed scales with hypothesis-set capacity. In Chapter 17’s ABC fit it can become an inferred parameter; here it is a sweep variable used to generate predictions.
  • max_dims = 3 is required because Sessions 2 and 3 contain rules with three or more feature dimensions (e.g., the “at least 2 of \(\{f_1, f_2, f_3\}\)” rule and the disjunctive Session 3 Nutritious rule). A max_dims = 2 grammar literally cannot express the true rules for those sessions. The relaxation is forced by the task structure, not by modelling preference. This is itself a cognitive claim — that humans in this task can entertain rules with three or more conjuncts, even if Chapter 14’s Kruschke-task agents could not.

What we commit to here: parameter sweeps over N_particles and max_dims are intentional, with cognitive content, and named at the point of use. The Chapter 14 commitments are not violated silently; they are relaxed explicitly.

Summary of Phase 0 commitments specific to this chapter. Five binary features, two decisions fitted independently, Kalman covariance scaled to the unit cube, rule particle filter is Model A (the cognitive theory), parameter sweeps deliberately span a wider range than Chapter 14’s defaults, and the rule definitions are verified against the data before any simulation runs. This chapter does no Stan fitting; the validation pipeline resumes in Chapter 17, where the same three model families are fitted to real data — including ABC for Model A.

get_true_categories <- function(features_matrix, session) {

  f2 <- features_matrix[, 2]
  f3 <- features_matrix[, 3]
  f4 <- features_matrix[, 4]
  f5 <- features_matrix[, 5]

    # Dangerous: Fat Legs (f1) AND Dots (f3)
    # Nutritious: Eyes on Stalks (f4)
    dangerous  <- as.integer(f1 == 1 & f3 == 1)
    nutritious <- as.integer(f4 == 1)

  } else if (session == 2) {
    # Dangerous: Eyes on Stalks (f4)
    # Nutritious: at least 2 of f1, f2, f3

    # Dangerous: Eyes (f4) AND Blue (f5)

    )

    stop("Invalid session number")
  }

  tibble(dangerous = dangerous, nutritious = nutritious)
}

17.3.1 Phase 0 Q3: Rule Verification (halt-on-fail)

We verify the category labels just generated by re-deriving them from the rule definitions and checking that they agree. This is the canonical place for the data-verification commitment from Phase 0 Q3. If any session’s rules fail to match the labels, the chapter halts here rather than propagating the inconsistency into the simulations.

Code 
verify_session_rules <- function(definitions_df) {
  recomputed <- definitions_df |>
    rowwise() |>
    mutate(
      dang_recomputed = case_when(
        session == 1 ~ as.integer(f1 == 1 & f3 == 1),
        session == 2 ~ as.integer(f4 == 1),
        session == 3 ~ as.integer(f4 == 1 & f5 == 1)
      ),
      nutr_recomputed = case_when(
        session == 1 ~ as.integer(f4 == 1),
        session == 2 ~ as.integer((f1 + f2 + f3) >= 2),
        session == 3 ~ as.integer(
          (f2 + f3 == 2) | (f1 == 1 & ((f4 + f5) >= 1) & ((f2 + f3) >= 1))
        )
      )
    ) |>
    ungroup()

  agreement <- recomputed |>
    group_by(session) |>
    summarise(
      dang_match = mean(dangerous  == dang_recomputed),
      nutr_match = mean(nutritious == nutr_recomputed),
      .groups    = "drop"
    )

  print(agreement)

  if (any(agreement$dang_match < 1) || any(agreement$nutr_match < 1)) {
    stop(
      "Rule verification failed: at least one session's labels do not match ",
      "their rule. Inspect the agreement table above and reconcile before ",
      "proceeding. This is the Phase 0 Q3 halt-on-fail check."
    )
  }
  cat("\nAll sessions pass rule verification (100% agreement).\n")
}

17.4 Per-Agent Session Schedule Generation

As established in Chapters 12–14, the three models are all path-dependent: the prediction at trial \(t\) depends on the sequence of stimuli observed on trials \(1, \ldots, t-1\) (through the GCM’s exemplar memory, the Kalman filter’s prototype state, or the particle filter’s weight distribution). Using the same trial ordering for every simulated agent tests model behavior on a single trajectory rather than the distribution of trajectories that real participants experience. Each agent therefore receives its own independently randomized trial order, generated from a deterministic per-agent seed.

Code 
# Generate one session schedule for one agent.
create_session_trials <- function(session_num, stimuli_definitions, seed) {
  set.seed(seed)
  session_stimuli <- stimuli_definitions |> filter(session == session_num)
  n_stim          <- nrow(session_stimuli)    # 32

  trial_blocks <- map_dfr(seq_len(n_blocks), function(b) {
    session_stimuli[sample(n_stim), ] |>
      mutate(block = b, trial_in_block = seq_len(n_stim))
  })

  trial_blocks |>
    mutate(trial_within_subject = seq_len(n()) ) |>
    dplyr::select(session, block, trial_within_subject,
                  stimulus_id, f1:f5, dangerous, nutritious)
}

17.5 Model Implementations

The three model functions below are adapted from Chapters 12–14 to handle five binary features and two independent decisions per trial. Each function accepts a pre-built obs_matrix and category_feedback vector for one decision dimension (dangerous or nutritious), and returns a vector of simulated binary responses.

17.5.1 1. Exemplar Model (GCM)

The GCM calculates summed similarity to stored exemplars for each category. It is called separately for the dangerous and nutritious decisions, since the two dimensions are learned independently (Phase 0 Q2). For binary features, the Manhattan distance reduces to a weighted Hamming distance — the sum of attention weights on dimensions where the test stimulus and the stored exemplar differ.

Code 
# ── GCM for one decision dimension ──────────────────────────────────────────
# w                : attention weights (length n_features, sums to 1)
# c                : sensitivity parameter (> 0, natural scale)
# obs_matrix       : (trials × features) matrix
# category_feedback: true labels (0 or 1) for this dimension
#
# Phase 0 Q1: with binary features in {0, 1}, the Manhattan distance
# sum(w * |stim - exemplar|) = sum of weights on dimensions where stimuli differ.
# This is a weighted Hamming distance.
gcm_single_decision <- function(w, c, obs_matrix, category_feedback,
                                quiet = TRUE) {
  n_trials   <- nrow(obs_matrix)
  n_features <- ncol(obs_matrix)
  response_probs       <- numeric(n_trials)
  exemplars_features   <- matrix(NA_real_, nrow = 0, ncol = n_features)
  exemplars_category   <- numeric(0)

  for (i in seq_len(n_trials)) {
    if (!quiet && i %% 20 == 0) cat("GCM Trial", i, "\n")

    stim  <- obs_matrix[i, ]
    n_seen <- nrow(exemplars_features)

    if (n_seen == 0 || length(unique(exemplars_category)) < 2) {
      prob_cat_1 <- 0.5
    } else {
      sims <- vapply(seq_len(n_seen), function(e) {
        d <- sum(w * abs(stim - exemplars_features[e, ]))
        exp(-c * d)
      }, numeric(1))

      s1 <- sum(sims[exemplars_category == 1])
      s0 <- sum(sims[exemplars_category == 0])
      denom <- s1 + s0
      prob_cat_1 <- if (denom > 1e-9) s1 / denom else 0.5
    }

    response_probs[i] <- pmax(1e-9, pmin(1 - 1e-9, prob_cat_1))

    exemplars_features  <- rbind(exemplars_features, stim)
    exemplars_category  <- c(exemplars_category, category_feedback[i])
  }

  rbinom(n_trials, 1, response_probs)
}

17.5.2 2. Prototype Model (Kalman Filter)

The Kalman filter tracks a running prototype (mean vector and covariance) for each category. Each decision dimension (dangerous, nutritious) gets its own pair of prototypes. Phase 0 Q1 specified that initial_sigma_diag = 1.0 is the appropriate value for the 5D binary setting; the function below uses 1.0 as its default, replacing Chapter 13’s 10.0 (which was calibrated for a 2D continuous feature space spanning \([1, 4]\) and would have been wildly over-dispersed in the unit cube).

Code 
# ── Multivariate Kalman update ───────────────────────────────────────────────
multivariate_kalman_update <- function(mu_prev, sigma_prev, observation, r_matrix) {
  mu_prev     <- as.numeric(mu_prev)
  observation <- as.numeric(observation)
  sigma_prev  <- as.matrix(sigma_prev)
  r_matrix    <- as.matrix(r_matrix)
  n_dim       <- length(mu_prev)
  I           <- diag(n_dim)

  S     <- sigma_prev + r_matrix
  S_inv <- tryCatch(solve(S), error = function(e) MASS::ginv(S))
  K     <- sigma_prev %*% S_inv

  mu_new    <- as.numeric(mu_prev + K %*% (observation - mu_prev))
  IK        <- I - K
  sigma_new <- IK %*% sigma_prev %*% t(IK) + K %*% r_matrix %*% t(K)
  sigma_new <- (sigma_new + t(sigma_new)) / 2   # enforce symmetry

  list(mu = mu_new, sigma = sigma_new)
}

# ── Kalman prototype agent for one decision dimension ────────────────────────
# Phase 0 Q1: initial_sigma_diag = 1.0 (NOT 10.0 from Ch. 13). Features are
# binary in {0, 1}, so a prior covariance of 1.0 covers the full unit cube.
prototype_kalman_single_decision <- function(r_value, obs_matrix,
                                             category_feedback,
                                             initial_mu         = NULL,
                                             initial_sigma_diag = 1.0,
                                             b                  = 0.5,
                                             quiet              = TRUE) {
  n_trials   <- nrow(obs_matrix)
  n_features <- ncol(obs_matrix)

  # Initial prototype mean: 0.5 (midpoint of binary feature range)
  mu_init <- if (is.null(initial_mu)) rep(0.5, n_features) else initial_mu
  proto0  <- list(mu = mu_init, sigma = diag(initial_sigma_diag, n_features))
  proto1  <- list(mu = mu_init, sigma = diag(initial_sigma_diag, n_features))
  r_mat   <- diag(r_value, n_features)
  response_probs <- numeric(n_trials)

  log_sum_exp <- function(v) {
    m <- max(v[is.finite(v)])
    if (!is.finite(m)) return(-Inf)
    m + log(sum(exp(v - m)))
  }

  for (i in seq_len(n_trials)) {
    if (!quiet && i %% 20 == 0) cat("Kalman Trial", i, "\n")
    x <- as.numeric(obs_matrix[i, ])

    log_p0 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = proto0$mu,
                       sigma = proto0$sigma + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    log_p1 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = proto1$mu,
                       sigma = proto1$sigma + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    log_p0b <- log_p0 + log(1 - b)
    log_p1b <- log_p1 + log(b)
    lse     <- log_sum_exp(c(log_p0b, log_p1b))
    prob1   <- if (is.finite(lse)) exp(log_p1b - lse) else b
    response_probs[i] <- pmax(1e-9, pmin(1 - 1e-9, prob1))

    # Update the prototype for the correct category
    if (category_feedback[i] == 1) {

17.5.3 3. Rule-Based Model — Model A (Particle Filter)

The particle filter maintains a weighted distribution over candidate rule hypotheses. This is Model A from Chapter 14 — the cognitive theory itself, with intractable likelihood, fitted via ABC (not HMC) when inference is required. In this chapter we run it forward only, to generate predictions.

For the five-feature alien task, thresholds are fixed at 0.5 (the midpoint of the binary feature range — see Phase 0 Q1), so rules effectively test whether a feature equals 0 or 1. Phase 0 Q5 noted that we relax Chapter 14’s max_dims = 2 commitment because Sessions 2 and 3 require rules with three or more dimensions; the simulation grid below sweeps max_dims ∈ {1, 2, 3} deliberately.

Code 
# ── Rule evaluation ──────────────────────────────────────────────────────────
evaluate_rule <- function(rule, stimulus) {
  tryCatch({
    eval_cond <- function(d, thresh, op) {
      if (d > length(stimulus)) return(sample(0:1, 1))
      switch(op, ">" = stimulus[d] > thresh, "<" = stimulus[d] < thresh, FALSE)
    }

    if (length(rule$dimensions) == 1) {
      condition_met <- eval_cond(rule$dimensions[1], rule$thresholds[1], rule$operations[1])

    } else if (length(rule$dimensions) == 2) {
      c1 <- eval_cond(rule$dimensions[1], rule$thresholds[1], rule$operations[1])
      c2 <- eval_cond(rule$dimensions[2], rule$thresholds[2], rule$operations[2])
      logic         <- if (is.null(rule$logic) || is.na(rule$logic)) "AND" else rule$logic
      condition_met <- switch(logic,
                              "AND" = c1 & c2,
                              "OR"  = c1 | c2,
                              c1 & c2)
    } else {
      # More than 2 dimensions: AND across all conditions.
      # Phase 0 Q5: max_dims > 2 is a deliberate relaxation of Ch. 14's commitment.
      results <- vapply(seq_along(rule$dimensions), function(k)
        eval_cond(rule$dimensions[k], rule$thresholds[k], rule$operations[k]),
        logical(1))
      condition_met <- all(results)
    }

    ifelse(condition_met, rule$prediction_if_true, 1L - rule$prediction_if_true)
  }, error = function(e) sample(0:1, 1))
}

# ── Rule generation ─────────────────────────────────────────────────────────
# For binary features, the only meaningful threshold is 0.5.
# Operators > 0.5 and < 0.5 thus correspond to "feature == 1" and "feature == 0".
generate_random_rule <- function(n_features, max_dims = 3) {
  num_dims   <- sample(seq_len(min(max_dims, n_features)), 1)
  dims_used  <- sort(sample(seq_len(n_features), size = num_dims, replace = FALSE))
  thresholds <- rep(0.5, num_dims)           # fixed midpoint for binary features
  operations <- sample(c(">", "<"), num_dims, replace = TRUE)
  list(
    dimensions        = dims_used,
    thresholds        = thresholds,
    operations        = operations,
    logic             = if (num_dims > 1) sample(c("AND", "OR"), 1) else NA,
    prediction_if_true = sample(0:1, 1)
  )
}

initialize_particles <- function(n_particles, n_features, max_dims) {
  particles <- vector("list", n_particles)
  for (i in seq_len(n_particles))
    particles[[i]] <- generate_random_rule(n_features, max_dims)
  list(particles = particles, weights = rep(1 / n_particles, n_particles))
}

update_weights <- function(particles, weights, stimulus, true_category, error_prob) {
  n    <- length(particles)
  likelihoods <- vapply(particles, function(p) {
    pred <- evaluate_rule(p, stimulus)
    if (pred == true_category) 1 - error_prob else error_prob
  }, numeric(1))
  new_w <- weights * likelihoods
  total <- sum(new_w)
  if (total > 1e-9) new_w / total else rep(1 / n, n)
}

resample_particles <- function(particles, weights) {
  n   <- length(particles)
  if (any(is.na(weights)) || sum(weights) <= 0 || any(weights < 0)) {
    return(list(particles = particles, weights = rep(1 / n, n)))
  }
  idx <- sample.int(n, size = n, replace = TRUE, prob = weights)
  list(particles = particles[idx], weights = rep(1 / n, n))
}

# ── Particle filter agent for one decision dimension ────────────────────────
# This is the forward simulator for Model A — the cognitive theory that
# Chapter 17 will fit to human data via ABC.
rule_particle_filter_single_decision <- function(n_particles, max_dims,
                                                 error_prob, obs_matrix,
                                                 category_feedback,
                                                 resample_threshold_factor = 0.5,
                                                 quiet = TRUE) {
  n_trials   <- nrow(obs_matrix)
  n_features <- ncol(obs_matrix)

  ps        <- initialize_particles(n_particles, n_features, max_dims)
  particles <- ps$particles
  weights   <- ps$weights
  response_probs <- numeric(n_trials)

  for (i in seq_len(n_trials)) {
    if (!quiet && i %% 20 == 0) cat("Rule Trial", i, "\n")
    stim <- as.numeric(obs_matrix[i, ])

    preds  <- vapply(particles, evaluate_rule, stimulus = stim, numeric(1))
    p1     <- sum(weights * ifelse(preds == 1, 1 - error_prob, error_prob))
    response_probs[i] <- pmax(1e-9, pmin(1 - 1e-9, p1))

    weights <- update_weights(particles, weights, stim, category_feedback[i], error_prob)

    ess <- 1 / sum(weights^2)
    if (!is.nan(ess) && ess < n_particles * resample_threshold_factor) {
      rs        <- resample_particles(particles, weights)
      particles <- rs$particles
      weights   <- rs$weights
    }
  }

  rbinom(n_trials, 1, response_probs)
}

17.8 Interpretation of Simulations

Examine the learning curves plotted above. The interpretations below are written as predictions: each one is a falsifiable claim about how the human data in Chapter 17 should look if the corresponding architecture is correct.

17.8.1 GCM (Exemplar)

How does performance differ between ‘Equal’ and ‘Optimal’ attention weights? Does optimal attention provide a significant advantage, especially in later sessions where rules become more complex?

The optimal weights place all weight on the features diagnostic for the underlying rule, and zero weight on irrelevant features. For the GCM, this is equivalent to computing similarity in the relevant subspace only. In Session 1, where the dangerous rule uses f1 and f3, a model with w = (1/2, 0, 1/2, 0, 0) will correctly ignore arm position and color when assessing danger. The equal-weight model treats all five features as equally diagnostic, which dilutes the signal from the truly relevant features and tends to produce slower or lower asymptotic learning.

Session 3 is particularly revealing: the nutritious rule is a complex disjunction over all five features, and the “optimal” weights for the GCM are set to uniform (since no single feature dominates). In this case, the GCM with equal and optimal weights should perform similarly, both relying on the full feature space to capture the disjunctive structure through exemplar storage.

The Chapter 17 prediction: when the multilevel GCM is fitted to human data, the population-level attention weights for each (session × decision) combination should concentrate on the diagnostic features for that combination. If they don’t — if humans put substantial weight on f2 for Session 1 dangerous, for example — then either the GCM’s selective-attention mechanism is wrong, or humans are using a non-GCM strategy that the GCM is absorbing into its attention weights as residual noise.

17.8.2 Kalman Filter (Prototype)

How does r_value (observation noise) affect learning speed and asymptotic accuracy across sessions?

The Kalman filter’s core challenge on this task is that several of the rules — especially the disjunctive Session 2 nutritious rule (at least 2 of 3 features active) and the complex Session 3 nutritious rule — do not map onto a unimodal category distribution in feature space. The prototype model’s assumption of a single Gaussian prototype per category is structurally misspecified for these rules.

An intermediate r_value often achieves the best balance between responsiveness and robustness, but even the best-parameterized Kalman filter may plateau below the performance achievable by models whose representational structure matches the task’s rule structure.

The Chapter 17 prediction: when the Kalman filter is fitted to human data, its ELPD should fall most sharply on the disjunctive sessions, and its posterior r_value should be biased upward to absorb the systematic misclassification from prototype averaging.

17.8.3 Rule Particle Filter (Model A)

How do the ‘simple’, ‘medium’, and ‘complex’ parameter settings affect performance across sessions?

The rule model is architecturally well-matched to Session 1 (where the true categories are defined by simple AND-logic rules on one or two features) and Session 2 (where dangerous is a 1D rule). The ‘simple’ setting (max_dims = 1) will likely succeed on the Session 1 nutritious rule (f4 only) and Session 2 dangerous rule (f4 only), but will fail on any rule requiring feature conjunctions. The ‘complex’ setting (max_dims = 3) can express the relevant multi-feature rules, at the cost of slower initial search through a larger rule space.

The ‘complex’ variant may show the characteristic sharp-improvement pattern of rule-based learning: broad uncertainty over many rules initially, then rapid concentration of weight on the correct rule once enough evidence accumulates. This produces learning curves with a distinctive “plateau then jump” shape, in contrast to the more gradual, monotonic learning curves of the GCM and Kalman filter.

The Chapter 17 prediction: when Model A is fitted to human data via ABC, the posterior over max_dims should concentrate on values \(\geq 2\) for Sessions 2 and 3, and the posterior over N_particles may distinguish “fast learners” (high \(N\), more parallel hypothesis testing) from “slow learners” (low \(N\)).

17.9 Observations from the Learning Curves

Based on the visualizations, we can draw several conclusions:

  1. Effect of Complexity (Sessions): Performance generally decreases or plateaus at a lower level as session complexity increases, reflecting the increased difficulty in identifying the underlying category structures.

  2. GCM Performance: The GCM with optimal weights typically outperforms the equal-weight GCM, especially in sessions 2 and 3. Even with optimal weights, the GCM may not reach perfect accuracy due to interference from similar but differently categorized exemplars. Its learning curve is often smooth and gradual — there is no discrete “discovery” event.

  3. Kalman Filter Performance: The model is sensitive to r_value. An intermediate value (e.g., \(r = 1.0\) or \(r = 2.0\)) often strikes a good balance. The Kalman filter is most likely to fall behind on disjunctive rules (Sessions 2 and 3 nutritious), where its single-prototype assumption is structurally mismatched to the task.

  4. Rule Particle Filter Performance (Model A): Highly dependent on max_dims. Allowing more complex rules is crucial in Sessions 2 and 3. Rule models can sometimes show sharper performance gains — the “aha moment” when the filter converges on the correct rule — but may also exhibit higher variance across agents because the search process is stochastic.

These forward simulations provide a baseline for understanding how different cognitive strategies fare on this task. Critically, the simulations isolate architectural differences between the models by holding parameters at fixed values. They do not tell us which model best describes actual human behavior — that requires fitting the models to individual participant data, which is the focus of Chapter 17.

  1. Qualitative plausibility: Do the simulated learning curves look broadly like what human participants might produce? If a model predicts near-perfect performance from trial 1 under reasonable parameters, something is wrong.

## Introduction

This chapter simulates the three categorization models (Exemplar/GCM, Prototype/Kalman Filter, Rule-Based/Particle Filter) on a task adapted from Tylén et al. (2023). In this task, participants (or models) learn to categorize visually complex "alien" stimuli based on feedback.

### Stimuli and Features

The aliens vary along five binary features:

* **f1 — Legs**: Slim (0) / Fat (1)
* **f2 — Arms**: Downwards (0) / Upwards (1)
* **f3 — Body Dots**: Absent (0) / Present (1)
* **f4 — Eyes**: Normal (0) / On Stalks (1)
* **f5 — Color**: Green (0) / Blue (1)

Each alien stimulus is a 5-dimensional binary vector (e.g., `c(0, 1, 0, 1, 1)` means slim legs, arms up, no dots, eyes on stalks, blue color). There are $2^5 = 32$ unique aliens.

### Categorization Dimensions and Task

Participants must react to each alien on two independent binary categorizations:

* **Dangerous**: Is the alien dangerous (1) or peaceful (0)?
* **Nutritious**: Is the alien nutritious (1) or non-nutritious (0)?

This results in four possible response combinations. The goal is to learn the underlying rules or patterns that determine each dimension based on the alien's features.

### Experimental Sessions and Complexity

The task involves three sessions with increasing rule complexity. Each session consists of three blocks where the 32 unique aliens are presented once per block in random order, with feedback provided after each choice.

N.B. the article also discusses a test session where participants are presented with new aliens. This is not implemented here since the novelty is in how the features are visually represented, and that change is transparent to our models.

### Category Rules per Session

Based on Tylén et al. (2023):

**Session 1 (Low Complexity):**

* Dangerous = f1 (Fat Legs) == 1 AND f3 (Dots) == 1
* Nutritious = f4 (Eyes on Stalks) == 1

**Session 2 (Intermediate Complexity):**

* Dangerous = f4 (Eyes on Stalks) == 1
* Nutritious = At least 2 out of f1, f2, f3 are 1 (i.e., f1 + f2 + f3 ≥ 2)

**Session 3 (High Complexity):**

* Dangerous = f4 (Eyes on Stalks) == 1 AND f5 (Blue Color) == 1
* Nutritious = (f2 + f3 == 2) OR (f1 == 1 AND (f4 + f5 ≥ 1) AND (f2 + f3 ≥ 1))

### Modeling Goal

We will simulate how the three categorization models learn the 'Dangerous' and 'Nutritious' categories across these three sessions. We are interested in comparing their learning trajectories (accuracy over trials) under different parameter assumptions — attention weights for the GCM, noise level for the Kalman filter, and rule complexity for the Particle Filter.

## Phase 0 for the Alien Game

Chapters 12–14 each opened with a Phase 0 interrogation that surfaced the structural commitments buried in the model code. This chapter inherits all of those commitments — and then adds three more, all stemming from the move to a new task. We surface them before any code runs, because the cleanest place to catch a hidden assumption is before it propagates into 800 lines of simulation.

> **Q1 — What changes when we move from 2D continuous features to 5D binary features?**
>
> Three concrete things, one per model family:
>
> * **For the GCM**: attention weights are now 5-vectors on the simplex rather than 2-vectors. The Manhattan distance $d(x, x_i) = \sum_m w_m |x_m - x_{i,m}|$ reduces (because all features are binary) to a *weighted Hamming distance* — the sum of weights on the dimensions where the two stimuli differ. This is a strict simplification of Chapter 12's continuous-feature setting, and it makes the GCM's decision rule especially transparent: similarity is "agreement on the features the model attends to."
>
> * **For the Kalman prototype model**: features now live in $\{0, 1\}$, so the prototypes live in the unit cube $[0, 1]^5$. Chapter 13 used `initial_sigma_diag = 10.0` for a 2D continuous space where features ranged over $[1, 4]$ — that prior covariance was already wide but defensible. **Carrying `10.0` into the 5D binary setting would put the prior covariance at roughly 100× the feature variance**, which is an absurdly diffuse prior on the unit cube. We commit here to `initial_sigma_diag = 1.0`, which puts the prior covariance on the same order of magnitude as the feature variance and matches Chapter 17's value when the same model is fitted to human data.
>
> * **For the rule particle filter**: thresholds are no longer drawn from a continuous range. The only meaningful threshold for a binary feature is 0.5, the midpoint between 0 and 1. The operators `>` and `<` then collapse to "feature equals 1" and "feature equals 0," respectively. The rule grammar from Chapter 14 — $r = (D, T, O, L, c)$ — survives this transition unchanged in form, but the threshold space is now a single point per dimension, making the grammar finite and discrete rather than combinatorial-times-continuous. This is *easier* for the cognitive theory to handle, not harder.
>
> **What we commit to here:** the dimensionality change is genuine and is named explicitly in the code; the Kalman initial covariance is fixed at `1.0`; the rule grammar's threshold space collapses to {0.5} per dimension.

> **Q2 — Why fit each decision dimension separately, rather than jointly?**
>
> The Alien Game asks two binary questions per trial — "Dangerous?" and "Nutritious?" — and earlier drafts of this chapter, like the implementation in this chapter, fitted each dimension independently. There are at least three alternatives:
>
> 1. **Joint 4-way categorical model.** A single 4-class outcome $\{(\text{D}, \text{N}), (\text{D}, \neg\text{N}), (\neg\text{D}, \text{N}), (\neg\text{D}, \neg\text{N})\}$, fitted with a softmax. This couples the two decisions and lets the model represent dependencies between them.
> 2. **Shared-attention joint model.** Two binary outcomes, but with a shared attention weight vector across the two decisions (the agent is constrained to attend to the same features for both).
> 3. **Independent fits** (what we do). Two binary outcomes fitted with separate parameters and separate sequential states.
>
> **What we commit to here:** independent fits. The cognitive claim is that the agent learns the two dimensions independently, with no transfer of attention or hypothesis content between them. This is *defensible* because the experimental design treats the two dimensions as separate questions with separate feedback channels, but it is a non-trivial claim — a real learner might allocate attention jointly, or notice a feature is informative for one dimension and try it for the other. We accept this commitment for pedagogical economy; relaxing it is a natural extension.

> **Q3 — Has the data been verified against the assumed rules?**
>
> Earlier drafts of this chapter contained a "bug fix note" tucked between two code blocks, mentioning that the Session 1 Dangerous rule had been corrected from `f2 & f3` to `f1 & f3` (Fat Legs, not Arms Up), and that a comment typo in the Session 2 Nutritious description had been fixed. **These corrections matter** — the GCM's optimal attention weights below depend on knowing the true rule. The honest place for this information is in a Phase 0 verification block that runs once and halts the chapter if any rule fails to match its expected definition.
>
> Below in §"Setup: Stimuli and True Category Labels," we generate the category labels under each session's rule and then verify the labels match what the rules should produce. The check is built into the data construction. If a future revision of the chapter changes the rule definitions or the feature mapping, the verification block will catch the inconsistency before it propagates.

> **Q4 — The rule particle filter in this chapter is Model A, not Model B.**
>
> Chapter 14 made a load-bearing distinction between two models with the same name:
>
> * **Model A** = the particle filter — the cognitive theory. Sequential, path-dependent, with a combinatorial rule space. Likelihood intractable in HMC. Fitted via Approximate Bayesian Computation (ABC).
> * **Model B** = a static marginalised mixture over a fixed candidate rule set, written in Stan. Closed-form likelihood. Conditionally exchangeable. A *biased proxy* for Model A.
>
> **The rule simulator in this chapter is Model A.** It is the cognitive theory itself, run forward to generate predictions. Model B does not appear in this chapter at all, because Model B has no learning curve and the chapter's purpose is to compare learning curves. When Chapter 17 fits the rule architecture to human data, it will fit *both* models — Model B via HMC for the proxy comparison, and Model A via ABC for the cognitive comparison — and report the gap between their posteriors as the structural-bias signature, exactly as Chapter 14 does on simulated data. This chapter's job is to set up the cognitive predictions Model A produces, so that the ABC-fitted Model A in Chapter 17 has something to be evaluated against.
>
> **What we commit to here:** the particle filter we run in this chapter is the cognitive theory. We do not run Model B. Comparison between the two happens in Chapter 17 on real data via ABC.

> **Q5 — Are we relaxing any of Chapter 14's structural commitments?**
>
> Yes. Chapter 14 committed to:
>
> * `N_PARTICLES_FIXED = 100` (a working-memory bound on hypothesis count)
> * `MAX_RULE_DIMS = 2` (rules use at most two feature conditions)
> * Operators `{>, <}` and logical connectives `{AND, OR}`
>
> This chapter intentionally varies `N_particles ∈ {50, 150, 250}` and `max_dims ∈ {1, 2, 3}` in the simulation grid below. Both relaxations are deliberate and have cognitive content:
>
> * **`N_particles` is varied** because we want to see how learning speed scales with hypothesis-set capacity. In Chapter 17's ABC fit it can become an inferred parameter; here it is a sweep variable used to *generate predictions*.
> * **`max_dims = 3` is required** because Sessions 2 and 3 contain rules with three or more feature dimensions (e.g., the "at least 2 of $\{f_1, f_2, f_3\}$" rule and the disjunctive Session 3 Nutritious rule). A `max_dims = 2` grammar literally cannot express the true rules for those sessions. The relaxation is forced by the task structure, not by modelling preference. **This is itself a cognitive claim** — that humans in this task can entertain rules with three or more conjuncts, even if Chapter 14's Kruschke-task agents could not.
>
> **What we commit to here:** parameter sweeps over `N_particles` and `max_dims` are intentional, with cognitive content, and named at the point of use. The Chapter 14 commitments are not violated *silently*; they are relaxed *explicitly*.

**Summary of Phase 0 commitments specific to this chapter.** Five binary features, two decisions fitted independently, Kalman covariance scaled to the unit cube, rule particle filter is Model A (the cognitive theory), parameter sweeps deliberately span a wider range than Chapter 14's defaults, and the rule definitions are verified against the data before any simulation runs. This chapter does no Stan fitting; the validation pipeline resumes in Chapter 17, where the same three model families are fitted to real data — including ABC for Model A.

## Model Implementations

The three model functions below are adapted from Chapters 12–14 to handle five binary features and two independent decisions per trial. Each function accepts a pre-built `obs_matrix` and `category_feedback` vector for one decision dimension (dangerous or nutritious), and returns a vector of simulated binary responses.

### 1. Exemplar Model (GCM)

The GCM calculates summed similarity to stored exemplars for each category. It is called separately for the dangerous and nutritious decisions, since the two dimensions are learned independently (Phase 0 Q2). For binary features, the Manhattan distance reduces to a weighted Hamming distance — the sum of attention weights on dimensions where the test stimulus and the stored exemplar differ.

```{r ch15_gcm}
# ── GCM for one decision dimension ──────────────────────────────────────────
# w                : attention weights (length n_features, sums to 1)
# c                : sensitivity parameter (> 0, natural scale)
# obs_matrix       : (trials × features) matrix
# category_feedback: true labels (0 or 1) for this dimension
#
# Phase 0 Q1: with binary features in {0, 1}, the Manhattan distance
# sum(w * |stim - exemplar|) = sum of weights on dimensions where stimuli differ.
# This is a weighted Hamming distance.
gcm_single_decision <- function(w, c, obs_matrix, category_feedback,
                                quiet = TRUE) {
  n_trials   <- nrow(obs_matrix)
  n_features <- ncol(obs_matrix)
  response_probs       <- numeric(n_trials)
  exemplars_features   <- matrix(NA_real_, nrow = 0, ncol = n_features)
  exemplars_category   <- numeric(0)

  for (i in seq_len(n_trials)) {
    if (!quiet && i %% 20 == 0) cat("GCM Trial", i, "\n")

    stim  <- obs_matrix[i, ]
    n_seen <- nrow(exemplars_features)

    if (n_seen == 0 || length(unique(exemplars_category)) < 2) {
      prob_cat_1 <- 0.5
    } else {
      sims <- vapply(seq_len(n_seen), function(e) {
        d <- sum(w * abs(stim - exemplars_features[e, ]))
        exp(-c * d)
      }, numeric(1))

      s1 <- sum(sims[exemplars_category == 1])
      s0 <- sum(sims[exemplars_category == 0])
      denom <- s1 + s0
      prob_cat_1 <- if (denom > 1e-9) s1 / denom else 0.5
    }

    response_probs[i] <- pmax(1e-9, pmin(1 - 1e-9, prob_cat_1))

    exemplars_features  <- rbind(exemplars_features, stim)
    exemplars_category  <- c(exemplars_category, category_feedback[i])
  }

  rbinom(n_trials, 1, response_probs)
}

cat("GCM function defined.\n")
```

### 2. Prototype Model (Kalman Filter)

The Kalman filter tracks a running prototype (mean vector and covariance) for each category. Each decision dimension (dangerous, nutritious) gets its own pair of prototypes. Phase 0 Q1 specified that `initial_sigma_diag = 1.0` is the appropriate value for the 5D binary setting; the function below uses 1.0 as its default, replacing Chapter 13's 10.0 (which was calibrated for a 2D continuous feature space spanning $[1, 4]$ and would have been wildly over-dispersed in the unit cube).

```{r ch15_kalman}
# ── Multivariate Kalman update ───────────────────────────────────────────────
multivariate_kalman_update <- function(mu_prev, sigma_prev, observation, r_matrix) {
  mu_prev     <- as.numeric(mu_prev)
  observation <- as.numeric(observation)
  sigma_prev  <- as.matrix(sigma_prev)
  r_matrix    <- as.matrix(r_matrix)
  n_dim       <- length(mu_prev)
  I           <- diag(n_dim)

  S     <- sigma_prev + r_matrix
  S_inv <- tryCatch(solve(S), error = function(e) MASS::ginv(S))
  K     <- sigma_prev %*% S_inv

  mu_new    <- as.numeric(mu_prev + K %*% (observation - mu_prev))
  IK        <- I - K
  sigma_new <- IK %*% sigma_prev %*% t(IK) + K %*% r_matrix %*% t(K)
  sigma_new <- (sigma_new + t(sigma_new)) / 2   # enforce symmetry

  list(mu = mu_new, sigma = sigma_new)
}

# ── Kalman prototype agent for one decision dimension ────────────────────────
# Phase 0 Q1: initial_sigma_diag = 1.0 (NOT 10.0 from Ch. 13). Features are
# binary in {0, 1}, so a prior covariance of 1.0 covers the full unit cube.
prototype_kalman_single_decision <- function(r_value, obs_matrix,
                                             category_feedback,
                                             initial_mu         = NULL,
                                             initial_sigma_diag = 1.0,
                                             b                  = 0.5,
                                             quiet              = TRUE) {
  n_trials   <- nrow(obs_matrix)
  n_features <- ncol(obs_matrix)

  # Initial prototype mean: 0.5 (midpoint of binary feature range)
  mu_init <- if (is.null(initial_mu)) rep(0.5, n_features) else initial_mu
  proto0  <- list(mu = mu_init, sigma = diag(initial_sigma_diag, n_features))
  proto1  <- list(mu = mu_init, sigma = diag(initial_sigma_diag, n_features))
  r_mat   <- diag(r_value, n_features)
  response_probs <- numeric(n_trials)

  log_sum_exp <- function(v) {
    m <- max(v[is.finite(v)])
    if (!is.finite(m)) return(-Inf)
    m + log(sum(exp(v - m)))
  }

  for (i in seq_len(n_trials)) {
    if (!quiet && i %% 20 == 0) cat("Kalman Trial", i, "\n")
    x <- as.numeric(obs_matrix[i, ])

    log_p0 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = proto0$mu,
                       sigma = proto0$sigma + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    log_p1 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = proto1$mu,
                       sigma = proto1$sigma + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    log_p0b <- log_p0 + log(1 - b)
    log_p1b <- log_p1 + log(b)
    lse     <- log_sum_exp(c(log_p0b, log_p1b))
    prob1   <- if (is.finite(lse)) exp(log_p1b - lse) else b
    response_probs[i] <- pmax(1e-9, pmin(1 - 1e-9, prob1))

    # Update the prototype for the correct category
    if (category_feedback[i] == 1) {
      proto1 <- multivariate_kalman_update(proto1$mu, proto1$sigma, x, r_mat)
    } else {
      proto0 <- multivariate_kalman_update(proto0$mu, proto0$sigma, x, r_mat)
    }
  }

  rbinom(n_trials, 1, response_probs)
}
```

### 3. Rule-Based Model — Model A (Particle Filter)

The particle filter maintains a weighted distribution over candidate rule hypotheses. **This is Model A from Chapter 14** — the cognitive theory itself, with intractable likelihood, fitted via ABC (not HMC) when inference is required. In this chapter we run it forward only, to generate predictions.

For the five-feature alien task, thresholds are fixed at 0.5 (the midpoint of the binary feature range — see Phase 0 Q1), so rules effectively test whether a feature equals 0 or 1. Phase 0 Q5 noted that we relax Chapter 14's `max_dims = 2` commitment because Sessions 2 and 3 require rules with three or more dimensions; the simulation grid below sweeps `max_dims ∈ {1, 2, 3}` deliberately.

```{r ch15_rules}
# ── Rule evaluation ──────────────────────────────────────────────────────────
evaluate_rule <- function(rule, stimulus) {
  tryCatch({
    eval_cond <- function(d, thresh, op) {
      if (d > length(stimulus)) return(sample(0:1, 1))
      switch(op, ">" = stimulus[d] > thresh, "<" = stimulus[d] < thresh, FALSE)
    }

    if (length(rule$dimensions) == 1) {
      condition_met <- eval_cond(rule$dimensions[1], rule$thresholds[1], rule$operations[1])

    } else if (length(rule$dimensions) == 2) {
      c1 <- eval_cond(rule$dimensions[1], rule$thresholds[1], rule$operations[1])
      c2 <- eval_cond(rule$dimensions[2], rule$thresholds[2], rule$operations[2])
      logic         <- if (is.null(rule$logic) || is.na(rule$logic)) "AND" else rule$logic
      condition_met <- switch(logic,
                              "AND" = c1 & c2,
                              "OR"  = c1 | c2,
                              c1 & c2)
    } else {
      # More than 2 dimensions: AND across all conditions.
      # Phase 0 Q5: max_dims > 2 is a deliberate relaxation of Ch. 14's commitment.
      results <- vapply(seq_along(rule$dimensions), function(k)
        eval_cond(rule$dimensions[k], rule$thresholds[k], rule$operations[k]),
        logical(1))
      condition_met <- all(results)
    }

    ifelse(condition_met, rule$prediction_if_true, 1L - rule$prediction_if_true)
  }, error = function(e) sample(0:1, 1))
}

# ── Rule generation ─────────────────────────────────────────────────────────
# For binary features, the only meaningful threshold is 0.5.
# Operators > 0.5 and < 0.5 thus correspond to "feature == 1" and "feature == 0".
generate_random_rule <- function(n_features, max_dims = 3) {
  num_dims   <- sample(seq_len(min(max_dims, n_features)), 1)
  dims_used  <- sort(sample(seq_len(n_features), size = num_dims, replace = FALSE))
  thresholds <- rep(0.5, num_dims)           # fixed midpoint for binary features
  operations <- sample(c(">", "<"), num_dims, replace = TRUE)
  list(
    dimensions        = dims_used,
    thresholds        = thresholds,
    operations        = operations,
    logic             = if (num_dims > 1) sample(c("AND", "OR"), 1) else NA,
    prediction_if_true = sample(0:1, 1)
  )
}

initialize_particles <- function(n_particles, n_features, max_dims) {
  particles <- vector("list", n_particles)
  for (i in seq_len(n_particles))
    particles[[i]] <- generate_random_rule(n_features, max_dims)
  list(particles = particles, weights = rep(1 / n_particles, n_particles))
}

update_weights <- function(particles, weights, stimulus, true_category, error_prob) {
  n    <- length(particles)
  likelihoods <- vapply(particles, function(p) {
    pred <- evaluate_rule(p, stimulus)
    if (pred == true_category) 1 - error_prob else error_prob
  }, numeric(1))
  new_w <- weights * likelihoods
  total <- sum(new_w)
  if (total > 1e-9) new_w / total else rep(1 / n, n)
}

resample_particles <- function(particles, weights) {
  n   <- length(particles)
  if (any(is.na(weights)) || sum(weights) <= 0 || any(weights < 0)) {
    return(list(particles = particles, weights = rep(1 / n, n)))
  }
  idx <- sample.int(n, size = n, replace = TRUE, prob = weights)
  list(particles = particles[idx], weights = rep(1 / n, n))
}

# ── Particle filter agent for one decision dimension ────────────────────────
# This is the forward simulator for Model A — the cognitive theory that
# Chapter 17 will fit to human data via ABC.
rule_particle_filter_single_decision <- function(n_particles, max_dims,
                                                 error_prob, obs_matrix,
                                                 category_feedback,
                                                 resample_threshold_factor = 0.5,
                                                 quiet = TRUE) {
  n_trials   <- nrow(obs_matrix)
  n_features <- ncol(obs_matrix)

  ps        <- initialize_particles(n_particles, n_features, max_dims)
  particles <- ps$particles
  weights   <- ps$weights
  response_probs <- numeric(n_trials)

  for (i in seq_len(n_trials)) {
    if (!quiet && i %% 20 == 0) cat("Rule Trial", i, "\n")
    stim <- as.numeric(obs_matrix[i, ])

    preds  <- vapply(particles, evaluate_rule, stimulus = stim, numeric(1))
    p1     <- sum(weights * ifelse(preds == 1, 1 - error_prob, error_prob))
    response_probs[i] <- pmax(1e-9, pmin(1 - 1e-9, p1))

    weights <- update_weights(particles, weights, stim, category_feedback[i], error_prob)

    ess <- 1 / sum(weights^2)
    if (!is.nan(ess) && ess < n_particles * resample_threshold_factor) {
      rs        <- resample_particles(particles, weights)
      particles <- rs$particles
      weights   <- rs$weights
    }
  }

  rbinom(n_trials, 1, response_probs)
}

cat("Rule Particle Filter (Model A) function defined.\n")
```

> on each simulated dataset, and accept the parameter draws whose summaries

## Interpretation of Simulations

Examine the learning curves plotted above. The interpretations below are written as predictions: each one is a falsifiable claim about how the human data in Chapter 17 should look if the corresponding architecture is correct.

### GCM (Exemplar)

How does performance differ between 'Equal' and 'Optimal' attention weights? Does optimal attention provide a significant advantage, especially in later sessions where rules become more complex?

The optimal weights place all weight on the features diagnostic for the underlying rule, and zero weight on irrelevant features. For the GCM, this is equivalent to computing similarity in the relevant subspace only. In Session 1, where the dangerous rule uses f1 and f3, a model with `w = (1/2, 0, 1/2, 0, 0)` will correctly ignore arm position and color when assessing danger. The equal-weight model treats all five features as equally diagnostic, which dilutes the signal from the truly relevant features and tends to produce slower or lower asymptotic learning.

Session 3 is particularly revealing: the nutritious rule is a complex disjunction over all five features, and the "optimal" weights for the GCM are set to uniform (since no single feature dominates). In this case, the GCM with equal and optimal weights should perform similarly, both relying on the full feature space to capture the disjunctive structure through exemplar storage.

**The Chapter 17 prediction**: when the multilevel GCM is fitted to human data, the population-level attention weights for each (session × decision) combination should concentrate on the diagnostic features for that combination. If they don't — if humans put substantial weight on f2 for Session 1 dangerous, for example — then either the GCM's selective-attention mechanism is wrong, or humans are using a non-GCM strategy that the GCM is absorbing into its attention weights as residual noise.

### Kalman Filter (Prototype)

How does `r_value` (observation noise) affect learning speed and asymptotic accuracy across sessions?

The Kalman filter's core challenge on this task is that several of the rules — especially the disjunctive Session 2 nutritious rule (at least 2 of 3 features active) and the complex Session 3 nutritious rule — do not map onto a unimodal category distribution in feature space. The prototype model's assumption of a single Gaussian prototype per category is structurally misspecified for these rules.

An intermediate `r_value` often achieves the best balance between responsiveness and robustness, but even the best-parameterized Kalman filter may plateau below the performance achievable by models whose representational structure matches the task's rule structure.

**The Chapter 17 prediction**: when the Kalman filter is fitted to human data, its ELPD should fall most sharply on the disjunctive sessions, and its posterior `r_value` should be biased upward to absorb the systematic misclassification from prototype averaging.

### Rule Particle Filter (Model A)

How do the 'simple', 'medium', and 'complex' parameter settings affect performance across sessions?

The rule model is architecturally well-matched to Session 1 (where the true categories are defined by simple AND-logic rules on one or two features) and Session 2 (where dangerous is a 1D rule). The 'simple' setting (max_dims = 1) will likely succeed on the Session 1 nutritious rule (f4 only) and Session 2 dangerous rule (f4 only), but will fail on any rule requiring feature conjunctions. The 'complex' setting (max_dims = 3) can express the relevant multi-feature rules, at the cost of slower initial search through a larger rule space.

The 'complex' variant may show the characteristic sharp-improvement pattern of rule-based learning: broad uncertainty over many rules initially, then rapid concentration of weight on the correct rule once enough evidence accumulates. This produces learning curves with a distinctive "plateau then jump" shape, in contrast to the more gradual, monotonic learning curves of the GCM and Kalman filter.

**The Chapter 17 prediction**: when Model A is fitted to human data via ABC, the posterior over `max_dims` should concentrate on values $\geq 2$ for Sessions 2 and 3, and the posterior over `N_particles` may distinguish "fast learners" (high $N$, more parallel hypothesis testing) from "slow learners" (low $N$).

## Observations from the Learning Curves

Based on the visualizations, we can draw several conclusions:

1. **Effect of Complexity (Sessions)**: Performance generally decreases or plateaus at a lower level as session complexity increases, reflecting the increased difficulty in identifying the underlying category structures.

2. **GCM Performance**: The GCM with optimal weights typically outperforms the equal-weight GCM, especially in sessions 2 and 3. Even with optimal weights, the GCM may not reach perfect accuracy due to interference from similar but differently categorized exemplars. Its learning curve is often smooth and gradual — there is no discrete "discovery" event.

3. **Kalman Filter Performance**: The model is sensitive to `r_value`. An intermediate value (e.g., $r = 1.0$ or $r = 2.0$) often strikes a good balance. The Kalman filter is most likely to fall behind on disjunctive rules (Sessions 2 and 3 nutritious), where its single-prototype assumption is structurally mismatched to the task.

4. **Rule Particle Filter Performance (Model A)**: Highly dependent on `max_dims`. Allowing more complex rules is crucial in Sessions 2 and 3. Rule models can sometimes show sharper performance gains — the "aha moment" when the filter converges on the correct rule — but may also exhibit higher variance across agents because the search process is stochastic.

These forward simulations provide a baseline for understanding how different cognitive strategies fare on this task. Critically, the simulations isolate architectural differences between the models by holding parameters at fixed values. They do **not** tell us which model best describes actual human behavior — that requires fitting the models to individual participant data, which is the focus of Chapter 17.
