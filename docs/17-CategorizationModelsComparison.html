📍 Where we are in the Bayesian modeling workflow: Chapters 12–14 each built, validated, and fitted a single-agent model to its own simulated dataset. Chapter 15 applied all three model families in simulation mode on the Alien Game. This chapter assembles the formal model comparison in two tracks:

Track 1 (Stan proxies, LOO/LFO-CV): We fit all three Stan models to the same common dataset, compute LOO scores, report ELPD differences with standard errors, visualize stacking weights, and compare decision boundaries. Crucially, the rule model in this track is Model B — the tractable Stan proxy from Chapter 14, not Model A (the particle filter). Comparing LOO for the real GCM against LOO for the rule proxy is an apples-to-oranges comparison, and the reader should know it.

Track 2 (NPE, cognitive theories on common footing): We fit all three cognitive theories — the GCM forward simulator, the Kalman prototype forward simulator, and the rule particle filter (Model A) — via Neural Posterior Estimation (NPE) using the sbi Python package. Each model gets its own trained inference network, validated with Simulation-Based Calibration Checks (SBC) + expected coverage, and compared via posterior predictive accuracy on the common dataset. This track resolves the apples-to-oranges problem: all three models are fitted as simulator-based models with no tractability advantage for any one of them. The gap between the two tracks’ conclusions is the chapter’s main empirical finding.

Chapter 17 applies this full two-track pipeline to human data from Tylén et al. (2023).

18.1 Three Perspectives on Categorization

Throughout this module, we have explored three major approaches to modeling categorization:

  1. Exemplar Models (GCM): Categories are represented by storing individual examples, and categorization decisions are based on similarity to these stored exemplars.

  2. Prototype Models (Kalman Filter): Categories are represented by their central tendencies or prototypes, which are dynamically updated as new examples are encountered.

  3. Rule-Based Models (Bayesian Particle Filter / Stan mixture): Categories are represented by explicit rules that define category boundaries, with rules discovered through hypothesis testing. Two distinct models exist for this architecture (see Ch. 14): Model A (the particle filter — the cognitive theory, fitted via NPE) and Model B (the Stan marginalised mixture — a tractable proxy). Track 1 uses Model B; Track 2 uses Model A.

Each approach captures different aspects of human categorization and makes distinct predictions about behavior. In this chapter, we systematically compare these models to understand their relative strengths, limitations, and complementary insights.

18.2 Phase 0: Structural Commitments for This Chapter

Before writing a single line of comparison code, we surface the load-bearing decisions that determine what this chapter’s comparison actually measures.

Q1 — Why does Track 1 use Model B (the Stan proxy) rather than Model A (the cognitive theory)?

Model A — the particle filter — has an intractable likelihood: no closed-form expression exists for \(P(y_{1:T} \mid \theta)\) because the joint probability requires integrating over an exponentially large space of rule trajectories. Stan’s HMC sampler requires evaluating and differentiating the log-likelihood, which is impossible for Model A. Model B is a tractable approximation: it marginalises over a fixed, finite set of candidate rules (\(R = 5\) in our Kruschke-task implementation), treating the evidence as conditionally independent given the error probability parameter. This gives a closed-form likelihood and makes HMC fitting possible.

What this means for the comparison: Track 1’s rule model is a proxy. Its LOO score measures how well a static mixture over 5 hand-picked rules predicts held-out responses — not how well the rule-learning cognitive theory predicts them. Comparing Model B’s ELPD against the GCM’s ELPD is, strictly speaking, an apples-to-oranges comparison. Track 2 resolves this by fitting all three cognitive theories via NPE, where tractability is not a constraint.

What we commit to here: Track 1 uses Model B and we flag this asymmetry explicitly at every point where the comparison is reported. The reader should treat Track 1’s rule model ELPD as a lower bound on what the cognitive theory (Model A) would achieve if it were tractable.

Q2 — What does “fair comparison” require, and does using GCM-generated data guarantee the GCM wins?

A fair model comparison requires: (a) the same observations \(y_{1:T}\) and stimulus sequences \(x_{1:T}\) entering every model’s likelihood; (b) the same cross-validation protocol for every model; (c) priors calibrated to the same scale. Our common dataset satisfies (a) and (c). Condition (b) is partially violated: LOO is technically valid for Model B (conditionally exchangeable) but only approximate for the GCM and Kalman filter (path-dependent). LFO-CV resolves this but is expensive; we defer full LFO-CV to Chapter 17.

Does GCM-generated data guarantee the GCM wins? No. Three failure modes exist: (i) numerical instability in the GCM’s sequential memory computation could produce poor log-likelihood values despite correct model specification; (ii) if the GCM’s prior is too diffuse, the posterior may not concentrate on the true parameters within 64 trials; (iii) the finite candidate-rule set of Model B may happen to contain a rule that is nearly optimal for the specific GCM trajectory we generated, letting Model B “win” despite being the wrong model. These are empirical questions; the comparison is designed to be informative, not guaranteed.

What we commit to here: we generate data from the GCM with known parameters, fit all three models, and report the comparison honestly — including the path-dependence caveat. If the GCM does not win, we diagnose why before concluding anything about the models.

Q3 — What does NPE add that ABC (from Ch. 14) doesn’t?

ABC (introduced in Chapter 14) requires hand-crafted summary statistics and a rejection threshold. Its accuracy depends on how well the chosen summaries capture the full likelihood. NPE (Neural Posterior Estimation) trains a neural network to directly estimate the posterior \(P(\theta \mid y)\) without summary statistics — it uses the full response vector as input. Key differences:

  • ABC is non-parametric and requires no training data, but scales poorly with parameter dimensionality and is sensitive to the choice of summary statistics.
  • NPE amortises the inference cost: after one training run (expensive), inference on new observations is essentially free. But it requires a large training set ($$10k simulations) and the posterior approximation is only as good as the training data coverage and the neural architecture.

In this chapter’s context, NPE is the right choice because we are comparing three models and want to run inference on a single common dataset. ABC would require separate tuning of summary statistics for each model. NPE trains one network per model and produces posteriors that are comparable on a common scale.

What we commit to here: we train NPE networks with \(n = 20{,}000\) simulations per model (set to a lower number in the default run; increase n_train for production). We validate each network with a Simulation-Based Calibration Checks (SBC) spot-check before using it for inference.

Q4 — Why is LOO exact for the rule model but approximate for the GCM and Kalman filter?

Standard PSIS-LOO estimates the ELPD for the prediction task: “given all observations except trial \(t\), how well does the model predict trial \(t\)?” This is well-defined if the likelihood of trial \(t\) does not depend on which other trials are included — i.e., the observations are conditionally exchangeable given the parameters.

  • Model B (Stan rule proxy): once error_prob is fixed, every trial’s likelihood is \(P(y_t \mid \theta) = (1-\varepsilon)^{[y_t = \hat{y}_t]} \varepsilon^{[y_t \neq \hat{y}_t]}\) — a function only of the current trial’s stimulus and response. No memory. Conditionally exchangeable. LOO is exact.

  • GCM and Kalman filter: the response probability at trial \(t\) depends on the entire history \(y_{1:t-1}\) through the exemplar memory (GCM) or Kalman filter state. Leaving out trial \(t\) in LOO conditions on a counterfactual memory state — one that was never built during training. The LOO predictive is for a prediction problem that does not correspond to any reachable experimental state. LOO is approximate.

LFO-CV leaves out a contiguous block of future trials and refits on the past, properly respecting temporal dependency. It is the correct tool for path-dependent models. We report approximate LOO here for comparability, but note every path-dependent ELPD as approximate and defer exact LFO-CV to Chapter 17.

What we commit to here: approximate LOO for GCM and Kalman, exact LOO for Model B. All ELPD differences involving the GCM or Kalman are flagged with the approximation caveat. Track 2’s NPE posterior predictive accuracy does not have this problem (no LOO; we evaluate predictive accuracy on the observed data under the posterior).

Q5 — What can go wrong with the NPE training, and how do we detect it?

Three main failure modes:

  1. Training data coverage failure: if the prior is too narrow to cover the true parameters, or too wide so that most simulations produce uninformative responses, the trained network will not generalise to the observed data. Detected by: poor SBC coverage, or posterior concentrating at the prior boundary.
  2. Neural architecture underfitting: if the network lacks capacity to represent the posterior, it will produce over-dispersed or biased posteriors. Detected by: SBC rank histograms that are U-shaped (over-dispersed) or hump-shaped (over-concentrated).
  3. Label/data mismatch: if the simulator and the real data use different conventions (e.g., 0-indexed vs. 1-indexed responses), the posterior will be nonsense. Detected by: posterior predictive accuracy near chance.

What we commit to here: we run a 200-iteration SBC spot-check on the GCM NPE network (the generating model for the common dataset) and plot rank histograms. Full 1000-iteration SBC for all three networks is in Chapter 17 where human data is analysed.

Summary of Phase 0 commitments. Track 1 uses Model B (the Stan proxy) for the rule architecture and flags this asymmetry everywhere. Track 2 uses NPE to fit all three cognitive theories on equal footing, validated with SBC. LOO is exact for Model B and approximate for the GCM and Kalman filter; LFO-CV is deferred to Chapter 17. The common dataset is GCM-generated, giving a known ground truth, but the GCM winning is not guaranteed and diagnosed if it fails.

18.4 Mathematical Comparison

18.4.1 Exemplar Model (GCM)

\[P(C_A \mid x) = \frac{\sum_{i \in A} \eta(x, x_i)}{\sum_{i \in A} \eta(x, x_i) + \sum_{j \in B} \eta(x, x_j)}\]

where \(\eta(x, x_i) = e^{-c \cdot d(x, x_i)}\) and \(d(x, x_i) = \sum_{m} w_m \lvert x_m - x_{i,m} \rvert\).

18.4.2 Prototype Model (Kalman Filter)

\[P(C_A \mid x) = \frac{\mathcal{N}(x \mid \mu_A,\, \Sigma_A + R)}{\mathcal{N}(x \mid \mu_A,\, \Sigma_A + R) + \mathcal{N}(x \mid \mu_B,\, \Sigma_B + R)}\]

where \(\mu_A, \Sigma_A\) are the running prototype mean and covariance for category A, and \(R = r \cdot I\) is the observation noise matrix.

18.4.3 Rule-Based Model — Model B (Stan marginalised mixture, not the particle filter)

⚠️ Track 1 structural-bias caveat. The model fitted below is Model B — the static, conditionally exchangeable mixture over a fixed candidate rule set from Chapter 14. It is not Model A (the particle filter), which is the actual cognitive theory. Model B has no learning dynamics, no resampling, no working-memory bound. Comparing Model B’s LOO against the GCM’s LOO compares a proxy against a real theory. Track 2 (§“NPE Comparison Under the Cognitive Theories”) resolves this by fitting all three cognitive theories via NPE on common footing.

\[P(C_A \mid x) = \sum_{r=1}^{R} P(C_A \mid x, \text{rule}_r) \cdot P(\text{rule}_r)\]

where \(P(C_A \mid x, \text{rule}_r) = 1 - \varepsilon\) if rule \(r\) predicts category A, and \(\varepsilon\) otherwise, and \(P(\text{rule}_r) = 1/R\) (uniform prior over the \(R\) candidate rules).

18.5 A Common Dataset for Fair Comparison

Chapters 12–14 each fitted their models to different simulated datasets. To compare models meaningfully we need them all estimated on the same data. A fair comparison requires identical observations \(y_{1:T}\) and stimulus sequences \(x_{1:T}\) entering each model’s likelihood.

We generate a fresh common dataset here by simulating one agent from the GCM with moderate parameters, then fitting all three Stan models to that agent’s responses. This design choice — using GCM-generated data — is deliberate: if the GCM generated the data, a well-functioning model comparison should prefer the GCM. If it does not, we have learned something about the models’ identifiability on this task (see Phase 0 Q2).

# ── GCM simulation function (from Ch. 12) ───────────────────────────────────
gcm_simulate <- function(w, log_c, bias, obs, cat_feedback, quiet = TRUE) {
  c_val     <- exp(log_c)
  ntrials   <- nrow(obs)
  nfeatures <- ncol(obs)
  prob_cat1 <- numeric(ntrials)
  sim_response <- numeric(ntrials)
  memory_obs  <- matrix(NA_real_, nrow = 0, ncol = nfeatures)
  memory_cat  <- numeric(0)
  for (i in seq_len(ntrials)) {
    stim <- as.numeric(obs[i, ])
    n_mem <- nrow(memory_obs)
    has0  <- any(memory_cat == 0)
    has1  <- any(memory_cat == 1)
    if (n_mem == 0 || !has0 || !has1) {
      p <- bias
    } else {
      sims <- vapply(seq_len(n_mem), function(e)
        exp(-c_val * sum(w * abs(stim - memory_obs[e, ]))), numeric(1))
      s1 <- sum(sims[memory_cat == 1])
      s0 <- sum(sims[memory_cat == 0])
      den <- s1 + s0
      p   <- if (den > 1e-9) bias * s1 / (bias * s1 + (1 - bias) * s0) else bias
    }
    prob_cat1[i]    <- pmax(1e-9, pmin(1 - 1e-9, p))
    sim_response[i] <- rbinom(1, 1, prob_cat1[i])
    memory_obs      <- rbind(memory_obs, stim)
    memory_cat      <- c(memory_cat, cat_feedback[i])
  }
  tibble(prob_cat1 = prob_cat1, sim_response = sim_response)
}

run_fit <- function(mod, data, seed, adapt_delta = 0.9,
                    iter_w = 1000, iter_s = 1500,
                    chains = 4) {
  pf <- tryCatch(
    mod$pathfinder(data = data, seed = seed, num_paths = 4, refresh = 0),

      NULL

    data            = data,
    init            = if (!is.null(pf)) pf else 0.5,

    iter_warmup     = iter_w,
    iter_sampling   = iter_s,
    refresh         = 300,
    adapt_delta     = adapt_delta
  )
}

18.7.1 The LFO-CV Caveat

Standard PSIS-LOO estimates the ELPD for a leave-one-out prediction task under the assumption that observations are conditionally exchangeable given the parameters. As established in Phase 0 Q4, this assumption holds for Model B but is violated for the GCM and Kalman filter. We report standard LOO here for comparability, with the understanding that ELPD differences between the GCM / Kalman models and the rule model are approximate and should be confirmed with LFO-CV before drawing firm conclusions. Chapter 17 performs this confirmation on human data.

18.9 Track 1 Summary

The LOO-based comparison above provides a first answer to “which model best predicts held-out responses on GCM-generated data.” But it has a load-bearing limitation: the rule model in Track 1 is Model B — a tractable proxy, not the cognitive theory. Model B has no learning dynamics, no resampling, no working-memory bound. Its LOO score tells you how well a static mixture over five hand-picked rules predicts the data; it does not tell you how well the rule-learning cognitive theory predicts the data. Comparing Model B’s LOO against the GCM’s LOO is comparing a proxy against a real theory, and interpreting the result as “the GCM beats the rule model” would be an over-claim.

Track 2 resolves this by fitting all three cognitive theories via Neural Posterior Estimation (NPE) on the same common dataset. The comparison is then on common footing: all three are simulator-based models, none has a tractability advantage.

This section fits the same common dataset under all three cognitive theories using NPE. The pipeline for each model is identical:

  1. Define the forward simulator (from Chapters 12–14).
  2. Hold the observed schedule fixed; sample parameters from the prior and simulate responses.
  3. Train an NPE inference network (neural spline flow with learned embedding).
  4. Validate with Simulation-Based Calibration Checks (SBC) + expected coverage on a calibration set.
  5. Perform inference on the observed data.
  6. Compute posterior predictive accuracy by sampling parameters from the posterior, running the simulator, and comparing to the observed responses.

# ── Kalman prototype simulator (from Ch. 13) ─────────────────────────────────
# theta_kalman = (log_r) — 1 parameter
simulate_kalman_for_npe <- function(theta) {
  r_val <- exp(theta[1])
  r_mat <- diag(r_val, 2)
  mu0 <- c(2.5, 2.5); mu1 <- c(2.5, 2.5)
  sig0 <- diag(10, 2); sig1 <- diag(10, 2)
  prob_cat1 <- numeric(n_trials_npe)

  for (i in seq_len(n_trials_npe)) {
    x <- npe_obs_mat[i, ]
    lp0 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = mu0, sigma = sig0 + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    lp1 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = mu1, sigma = sig1 + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    if (all(is.finite(c(lp0, lp1)))) {
      lse <- log(exp(lp0) + exp(lp1))
      prob_cat1[i] <- exp(lp1 - lse)
    } else {
      prob_cat1[i] <- 0.5
    }
    prob_cat1[i] <- pmax(1e-9, pmin(1 - 1e-9, prob_cat1[i]))

    cat_i <- npe_cat_one[i]
    if (cat_i == 1) {
      S <- sig1 + r_mat; K <- sig1 %*% solve(S)
      mu1  <- as.numeric(mu1 + K %*% (x - mu1))
      IK   <- diag(2) - K
      sig1 <- (IK %*% sig1 %*% t(IK) + K %*% r_mat %*% t(K))
      sig1 <- (sig1 + t(sig1)) / 2
    } else {
      S <- sig0 + r_mat; K <- sig0 %*% solve(S)
      mu0  <- as.numeric(mu0 + K %*% (x - mu0))
      IK   <- diag(2) - K
      sig0 <- (IK %*% sig0 %*% t(IK) + K %*% r_mat %*% t(K))
      sig0 <- (sig0 + t(sig0)) / 2
    }
  }
  rbinom(n_trials_npe, 1, prob_cat1)
}

.evaluate_rule_npe <- function(rule, stimulus) {
  tryCatch({
    eval_cond <- function(d, thresh, op)
      switch(op, ">" = stimulus[d] > thresh, "<" = stimulus[d] < thresh, FALSE)
    if (length(rule$dims) == 1) {
      cond <- eval_cond(rule$dims[1], rule$thresh[1], rule$ops[1])
    } else {
      c1 <- eval_cond(rule$dims[1], rule$thresh[1], rule$ops[1])
      c2 <- eval_cond(rule$dims[2], rule$thresh[2], rule$ops[2])
      cond <- if (rule$logic == "AND") c1 & c2 else c1 | c2
    }
    ifelse(cond, rule$pred, 1L - rule$pred)
  }, error = function(e) sample(0:1, 1))
}

.gen_rule_npe <- function(nf = 2, md = MAX_RULE_DIMS) {
  nd <- sample(seq_len(min(md, nf)), 1)
  dm <- sort(sample(seq_len(nf), nd, replace = FALSE))
  tr <- runif(nd, 1, 4)
  list(dims = dm, thresh = tr, ops = sample(c(">", "<"), nd, replace = TRUE),
       logic = if (nd > 1) sample(c("AND", "OR"), 1) else "AND",
       pred  = sample(0:1, 1))
}

simulate_rule_for_npe <- function(theta) {
  eps    <- plogis(theta[1])
  n_part <- max(10L, as.integer(round(exp(theta[2]))))
  parts  <- lapply(seq_len(n_part), function(i) .gen_rule_npe())
  wts    <- rep(1 / n_part, n_part)
  probs  <- numeric(n_trials_npe)

  for (i in seq_len(n_trials_npe)) {
    x  <- as.numeric(npe_obs_mat[i, ])
    pd <- vapply(parts, .evaluate_rule_npe, stimulus = x, numeric(1))
    probs[i] <- pmax(1e-9, pmin(1 - 1e-9,
      sum(wts * ifelse(pd == 1, 1 - eps, eps))))

    lk  <- vapply(parts, function(p) {
      pr <- .evaluate_rule_npe(p, x)
      if (pr == npe_cat_one[i]) 1 - eps else eps
    }, numeric(1))
    nw  <- wts * lk; tot <- sum(nw)
    wts <- if (tot > 1e-9) nw / tot else rep(1 / n_part, n_part)

    ess <- 1 / sum(wts^2)
    if (!is.nan(ess) && ess < n_part * RESAMPLE_THRESHOLD) {
      idx   <- sample.int(n_part, n_part, replace = TRUE, prob = wts)
      parts <- parts[idx]
      wts   <- rep(1 / n_part, n_part)
    }
  }
  rbinom(n_trials_npe, 1, probs)
}

train_npe_model <- function(simulator_fn, prior_low, prior_high, n_params,
                            model_name, n_sims = n_train) {
  cache_file <- sprintf("simdata/W16_npe_%s_train.rds", model_name)

  if (regenerate_fits || !file.exists(cache_file)) {
    cat("Generating", n_sims, "training simulations for", model_name, "...\n")
    set.seed(2026)
    thetas <- matrix(runif(n_sims * n_params,
                           min = prior_low, max = prior_high),
                     ncol = n_params, byrow = TRUE)
    responses <- future_map(seq_len(n_sims), function(i) {
      simulator_fn(thetas[i, ])
    }, .options = furrr_options(seed = TRUE), .progress = TRUE)
    y_mat <- do.call(rbind, responses)
    saveRDS(list(thetas = thetas, responses = y_mat), cache_file)
  } else {
    cached <- readRDS(cache_file)
    thetas <- cached$thetas
    y_mat  <- cached$responses
  }

  theta_t <- torch$tensor(thetas, dtype = torch$float32)
  x_t     <- torch$tensor(y_mat,  dtype = torch$float32)

  sbi_prior <- utils$BoxUniform(
    low  = torch$tensor(prior_low,  dtype = torch$float32),
    high = torch$tensor(prior_high, dtype = torch$float32)
  )

  cat("Training NPE for", model_name, "...\n")
  inference_obj <- infer$SNPE(
    prior = sbi_prior,
    density_estimator = utils$posterior_nn(
      model = "nsf", hidden_features = 64L, num_transforms = 5L,
      embedding_net = torch$nn$Sequential(
        torch$nn$Linear(as.integer(ncol(y_mat)), 128L),
        torch$nn$ReLU(),
        torch$nn$Linear(128L, 64L),
        torch$nn$ReLU(),
        torch$nn$Linear(64L, 32L)
      )
    )
  )
  inference_obj$append_simulations(theta_t, x_t)
  density_est <- inference_obj$train(
    training_batch_size = 256L, learning_rate = 5e-4,
    show_train_summary = TRUE
  )
  posterior <- inference_obj$build_posterior(density_est)
  list(posterior = posterior, thetas = thetas, responses = y_mat)
}

## Three Perspectives on Categorization

Throughout this module, we have explored three major approaches to modeling categorization:

1. **Exemplar Models (GCM)**: Categories are represented by storing individual examples, and categorization decisions are based on similarity to these stored exemplars.

2. **Prototype Models (Kalman Filter)**: Categories are represented by their central tendencies or prototypes, which are dynamically updated as new examples are encountered.

3. **Rule-Based Models (Bayesian Particle Filter / Stan mixture)**: Categories are represented by explicit rules that define category boundaries, with rules discovered through hypothesis testing. **Two distinct models exist for this architecture** (see Ch. 14): Model A (the particle filter — the cognitive theory, fitted via NPE) and Model B (the Stan marginalised mixture — a tractable proxy). Track 1 uses Model B; Track 2 uses Model A.

Each approach captures different aspects of human categorization and makes distinct predictions about behavior. In this chapter, we systematically compare these models to understand their relative strengths, limitations, and complementary insights.

## Phase 0: Structural Commitments for This Chapter

Before writing a single line of comparison code, we surface the load-bearing decisions that determine what this chapter's comparison actually measures.

> **Q1 — Why does Track 1 use Model B (the Stan proxy) rather than Model A (the cognitive theory)?**
>
> Model A — the particle filter — has an intractable likelihood: no closed-form expression exists for $P(y_{1:T} \mid \theta)$ because the joint probability requires integrating over an exponentially large space of rule trajectories. Stan's HMC sampler requires evaluating and differentiating the log-likelihood, which is impossible for Model A. Model B is a tractable approximation: it marginalises over a **fixed, finite** set of candidate rules ($R = 5$ in our Kruschke-task implementation), treating the evidence as conditionally independent given the error probability parameter. This gives a closed-form likelihood and makes HMC fitting possible.
>
> **What this means for the comparison**: Track 1's rule model is a *proxy*. Its LOO score measures how well a static mixture over 5 hand-picked rules predicts held-out responses — not how well the rule-learning cognitive theory predicts them. Comparing Model B's ELPD against the GCM's ELPD is, strictly speaking, an apples-to-oranges comparison. Track 2 resolves this by fitting all three cognitive theories via NPE, where tractability is not a constraint.
>
> **What we commit to here:** Track 1 uses Model B and we flag this asymmetry explicitly at every point where the comparison is reported. The reader should treat Track 1's rule model ELPD as a lower bound on what the cognitive theory (Model A) would achieve if it were tractable.

> **Q2 — What does "fair comparison" require, and does using GCM-generated data guarantee the GCM wins?**
>
> A fair model comparison requires: (a) the same observations $y_{1:T}$ and stimulus sequences $x_{1:T}$ entering every model's likelihood; (b) the same cross-validation protocol for every model; (c) priors calibrated to the same scale. Our common dataset satisfies (a) and (c). Condition (b) is partially violated: LOO is technically valid for Model B (conditionally exchangeable) but only approximate for the GCM and Kalman filter (path-dependent). LFO-CV resolves this but is expensive; we defer full LFO-CV to Chapter 17.
>
> **Does GCM-generated data guarantee the GCM wins?** No. Three failure modes exist: (i) numerical instability in the GCM's sequential memory computation could produce poor log-likelihood values despite correct model specification; (ii) if the GCM's prior is too diffuse, the posterior may not concentrate on the true parameters within 64 trials; (iii) the finite candidate-rule set of Model B may happen to contain a rule that is nearly optimal for the specific GCM trajectory we generated, letting Model B "win" despite being the wrong model. These are empirical questions; the comparison is designed to be informative, not guaranteed.
>
> **What we commit to here:** we generate data from the GCM with known parameters, fit all three models, and report the comparison honestly — including the path-dependence caveat. If the GCM does not win, we diagnose why before concluding anything about the models.

> **Q3 — What does NPE add that ABC (from Ch. 14) doesn't?**
>
> ABC (introduced in Chapter 14) requires hand-crafted summary statistics and a rejection threshold. Its accuracy depends on how well the chosen summaries capture the full likelihood. NPE (Neural Posterior Estimation) trains a neural network to directly estimate the posterior $P(\theta \mid y)$ without summary statistics — it uses the full response vector as input. Key differences:
>
> * **ABC** is non-parametric and requires no training data, but scales poorly with parameter dimensionality and is sensitive to the choice of summary statistics.
> * **NPE** amortises the inference cost: after one training run (expensive), inference on new observations is essentially free. But it requires a large training set ($\geq$10k simulations) and the posterior approximation is only as good as the training data coverage and the neural architecture.
>
> In this chapter's context, NPE is the right choice because we are comparing *three* models and want to run inference on a single common dataset. ABC would require separate tuning of summary statistics for each model. NPE trains one network per model and produces posteriors that are comparable on a common scale.
>
> **What we commit to here:** we train NPE networks with $n = 20{,}000$ simulations per model (set to a lower number in the default run; increase `n_train` for production). We validate each network with a Simulation-Based Calibration Checks (SBC) spot-check before using it for inference.

> **Q4 — Why is LOO exact for the rule model but approximate for the GCM and Kalman filter?**
>
> Standard PSIS-LOO estimates the ELPD for the prediction task: "given all observations *except* trial $t$, how well does the model predict trial $t$?" This is well-defined if the likelihood of trial $t$ does not depend on which other trials are included — i.e., the observations are **conditionally exchangeable** given the parameters.
>
> * **Model B** (Stan rule proxy): once `error_prob` is fixed, every trial's likelihood is $P(y_t \mid \theta) = (1-\varepsilon)^{[y_t = \hat{y}_t]} \varepsilon^{[y_t \neq \hat{y}_t]}$ — a function only of the current trial's stimulus and response. No memory. Conditionally exchangeable. LOO is exact.
>
> * **GCM and Kalman filter**: the response probability at trial $t$ depends on the entire history $y_{1:t-1}$ through the exemplar memory (GCM) or Kalman filter state. Leaving out trial $t$ in LOO conditions on a counterfactual memory state — one that was never built during training. The LOO predictive is for a prediction problem that does not correspond to any reachable experimental state. LOO is approximate.
>
> LFO-CV leaves out a contiguous block of *future* trials and refits on the past, properly respecting temporal dependency. It is the correct tool for path-dependent models. We report approximate LOO here for comparability, but note every path-dependent ELPD as approximate and defer exact LFO-CV to Chapter 17.
>
> **What we commit to here:** approximate LOO for GCM and Kalman, exact LOO for Model B. All ELPD differences involving the GCM or Kalman are flagged with the approximation caveat. Track 2's NPE posterior predictive accuracy does not have this problem (no LOO; we evaluate predictive accuracy on the observed data under the posterior).

> **Q5 — What can go wrong with the NPE training, and how do we detect it?**
>
> Three main failure modes:
>
> 1. **Training data coverage failure**: if the prior is too narrow to cover the true parameters, or too wide so that most simulations produce uninformative responses, the trained network will not generalise to the observed data. Detected by: poor SBC coverage, or posterior concentrating at the prior boundary.
> 2. **Neural architecture underfitting**: if the network lacks capacity to represent the posterior, it will produce over-dispersed or biased posteriors. Detected by: SBC rank histograms that are U-shaped (over-dispersed) or hump-shaped (over-concentrated).
> 3. **Label/data mismatch**: if the simulator and the real data use different conventions (e.g., 0-indexed vs. 1-indexed responses), the posterior will be nonsense. Detected by: posterior predictive accuracy near chance.
>
> **What we commit to here:** we run a 200-iteration SBC spot-check on the GCM NPE network (the generating model for the common dataset) and plot rank histograms. Full 1000-iteration SBC for all three networks is in Chapter 17 where human data is analysed.

**Summary of Phase 0 commitments.** Track 1 uses Model B (the Stan proxy) for the rule architecture and flags this asymmetry everywhere. Track 2 uses NPE to fit all three cognitive theories on equal footing, validated with SBC. LOO is exact for Model B and approximate for the GCM and Kalman filter; LFO-CV is deferred to Chapter 17. The common dataset is GCM-generated, giving a known ground truth, but the GCM winning is not guaranteed and diagnosed if it fails.

## Mathematical Comparison

### Exemplar Model (GCM)

$$P(C_A \mid x) = \frac{\sum_{i \in A} \eta(x, x_i)}{\sum_{i \in A} \eta(x, x_i) + \sum_{j \in B} \eta(x, x_j)}$$

where $\eta(x, x_i) = e^{-c \cdot d(x, x_i)}$ and $d(x, x_i) = \sum_{m} w_m \lvert x_m - x_{i,m} \rvert$.

### Prototype Model (Kalman Filter)

$$P(C_A \mid x) = \frac{\mathcal{N}(x \mid \mu_A,\, \Sigma_A + R)}{\mathcal{N}(x \mid \mu_A,\, \Sigma_A + R) + \mathcal{N}(x \mid \mu_B,\, \Sigma_B + R)}$$

where $\mu_A, \Sigma_A$ are the running prototype mean and covariance for category A, and $R = r \cdot I$ is the observation noise matrix.

### Rule-Based Model — Model B (Stan marginalised mixture, *not* the particle filter)

> **⚠️ Track 1 structural-bias caveat.** The model fitted below is **Model B** —
> the static, conditionally exchangeable mixture over a fixed candidate rule
> set from Chapter 14. It is *not* Model A (the particle filter), which is the
> actual cognitive theory. Model B has no learning dynamics, no resampling, no
> working-memory bound. Comparing Model B's LOO against the GCM's LOO
> compares a proxy against a real theory. Track 2 (§"NPE Comparison Under the
> Cognitive Theories") resolves this by fitting all three cognitive theories
> via NPE on common footing.

$$P(C_A \mid x) = \sum_{r=1}^{R} P(C_A \mid x, \text{rule}_r) \cdot P(\text{rule}_r)$$

where $P(C_A \mid x, \text{rule}_r) = 1 - \varepsilon$ if rule $r$ predicts category A, and $\varepsilon$ otherwise, and $P(\text{rule}_r) = 1/R$ (uniform prior over the $R$ candidate rules).

```{r ch16_common_data}
# ── GCM simulation function (from Ch. 12) ───────────────────────────────────
gcm_simulate <- function(w, log_c, bias, obs, cat_feedback, quiet = TRUE) {
  c_val     <- exp(log_c)
  ntrials   <- nrow(obs)
  nfeatures <- ncol(obs)
  prob_cat1 <- numeric(ntrials)
  sim_response <- numeric(ntrials)
  memory_obs  <- matrix(NA_real_, nrow = 0, ncol = nfeatures)
  memory_cat  <- numeric(0)
  for (i in seq_len(ntrials)) {
    stim <- as.numeric(obs[i, ])
    n_mem <- nrow(memory_obs)
    has0  <- any(memory_cat == 0)
    has1  <- any(memory_cat == 1)
    if (n_mem == 0 || !has0 || !has1) {
      p <- bias
    } else {
      sims <- vapply(seq_len(n_mem), function(e)
        exp(-c_val * sum(w * abs(stim - memory_obs[e, ]))), numeric(1))
      s1 <- sum(sims[memory_cat == 1])
      s0 <- sum(sims[memory_cat == 0])
      den <- s1 + s0
      p   <- if (den > 1e-9) bias * s1 / (bias * s1 + (1 - bias) * s0) else bias
    }
    prob_cat1[i]    <- pmax(1e-9, pmin(1 - 1e-9, p))
    sim_response[i] <- rbinom(1, 1, prob_cat1[i])
    memory_obs      <- rbind(memory_obs, stim)
    memory_cat      <- c(memory_cat, cat_feedback[i])
  }
  tibble(prob_cat1 = prob_cat1, sim_response = sim_response)
}
```

## Track 1 Summary

The LOO-based comparison above provides a first answer to "which model best predicts held-out responses on GCM-generated data." But it has a load-bearing limitation: **the rule model in Track 1 is Model B — a tractable proxy, not the cognitive theory.** Model B has no learning dynamics, no resampling, no working-memory bound. Its LOO score tells you how well a static mixture over five hand-picked rules predicts the data; it does *not* tell you how well the rule-learning cognitive theory predicts the data. Comparing Model B's LOO against the GCM's LOO is comparing a proxy against a real theory, and interpreting the result as "the GCM beats the rule model" would be an over-claim.

Track 2 resolves this by fitting all three cognitive theories via Neural Posterior Estimation (NPE) on the same common dataset. The comparison is then on common footing: all three are simulator-based models, none has a tractability advantage.

## Track 2: NPE Comparison Under the Cognitive Theories

> **📦 Python dependency.** Track 2 requires the `sbi` Python package (accessed
> via `reticulate`). Install with: `pip install sbi torch`. All Track 2 code
> blocks are gated behind `run_npe_track <- FALSE`; set to `TRUE` to run.
> Training 20,000 simulations per model takes 5–15 minutes per model on a
> modern CPU; use a GPU for the neural network training step if available.

This section fits the same common dataset under all three cognitive theories using NPE. The pipeline for each model is identical:

1. Define the forward simulator (from Chapters 12–14).
2. Hold the observed schedule fixed; sample parameters from the prior and simulate responses.
3. Train an NPE inference network (neural spline flow with learned embedding).
4. Validate with Simulation-Based Calibration Checks (SBC) + expected coverage on a calibration set.
5. Perform inference on the observed data.
6. Compute posterior predictive accuracy by sampling parameters from the posterior, running the simulator, and comparing to the observed responses.

```{r ch16_npe_simulators, eval=run_npe_track}
simulate_gcm_for_npe <- function(theta) {
  w1   <- plogis(theta[1])
  w    <- c(w1, 1 - w1)
  lc   <- theta[2]
  bias <- plogis(theta[3])
  sim  <- gcm_simulate(w, lc, bias, npe_obs_mat, npe_cat_one)
  as.numeric(sim$sim_response)
}

# ── Kalman prototype simulator (from Ch. 13) ─────────────────────────────────
# theta_kalman = (log_r) — 1 parameter
simulate_kalman_for_npe <- function(theta) {
  r_val <- exp(theta[1])
  r_mat <- diag(r_val, 2)
  mu0 <- c(2.5, 2.5); mu1 <- c(2.5, 2.5)
  sig0 <- diag(10, 2); sig1 <- diag(10, 2)
  prob_cat1 <- numeric(n_trials_npe)

  for (i in seq_len(n_trials_npe)) {
    x <- npe_obs_mat[i, ]
    lp0 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = mu0, sigma = sig0 + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    lp1 <- tryCatch(
      mvtnorm::dmvnorm(x, mean = mu1, sigma = sig1 + r_mat, log = TRUE),
      error = function(e) -Inf
    )
    if (all(is.finite(c(lp0, lp1)))) {
      lse <- log(exp(lp0) + exp(lp1))
      prob_cat1[i] <- exp(lp1 - lse)
    } else {
      prob_cat1[i] <- 0.5
    }
    prob_cat1[i] <- pmax(1e-9, pmin(1 - 1e-9, prob_cat1[i]))

    cat_i <- npe_cat_one[i]
    if (cat_i == 1) {
      S <- sig1 + r_mat; K <- sig1 %*% solve(S)
      mu1  <- as.numeric(mu1 + K %*% (x - mu1))
      IK   <- diag(2) - K
      sig1 <- (IK %*% sig1 %*% t(IK) + K %*% r_mat %*% t(K))
      sig1 <- (sig1 + t(sig1)) / 2
    } else {
      S <- sig0 + r_mat; K <- sig0 %*% solve(S)
      mu0  <- as.numeric(mu0 + K %*% (x - mu0))
      IK   <- diag(2) - K
      sig0 <- (IK %*% sig0 %*% t(IK) + K %*% r_mat %*% t(K))
      sig0 <- (sig0 + t(sig0)) / 2
    }
  }
  rbinom(n_trials_npe, 1, prob_cat1)
}

# theta_rule = (logit_eps, log_n_particles) — 2 parameters

.evaluate_rule_npe <- function(rule, stimulus) {
  tryCatch({
    eval_cond <- function(d, thresh, op)
      switch(op, ">" = stimulus[d] > thresh, "<" = stimulus[d] < thresh, FALSE)
    if (length(rule$dims) == 1) {
      cond <- eval_cond(rule$dims[1], rule$thresh[1], rule$ops[1])
    } else {
      c1 <- eval_cond(rule$dims[1], rule$thresh[1], rule$ops[1])
      c2 <- eval_cond(rule$dims[2], rule$thresh[2], rule$ops[2])
      cond <- if (rule$logic == "AND") c1 & c2 else c1 | c2
    }
    ifelse(cond, rule$pred, 1L - rule$pred)
  }, error = function(e) sample(0:1, 1))
}

.gen_rule_npe <- function(nf = 2, md = MAX_RULE_DIMS) {
  nd <- sample(seq_len(min(md, nf)), 1)
  dm <- sort(sample(seq_len(nf), nd, replace = FALSE))
  tr <- runif(nd, 1, 4)
  list(dims = dm, thresh = tr, ops = sample(c(">", "<"), nd, replace = TRUE),
       logic = if (nd > 1) sample(c("AND", "OR"), 1) else "AND",
       pred  = sample(0:1, 1))
}

simulate_rule_for_npe <- function(theta) {
  eps    <- plogis(theta[1])
  n_part <- max(10L, as.integer(round(exp(theta[2]))))
  parts  <- lapply(seq_len(n_part), function(i) .gen_rule_npe())
  wts    <- rep(1 / n_part, n_part)
  probs  <- numeric(n_trials_npe)

  for (i in seq_len(n_trials_npe)) {
    x  <- as.numeric(npe_obs_mat[i, ])
    pd <- vapply(parts, .evaluate_rule_npe, stimulus = x, numeric(1))
    probs[i] <- pmax(1e-9, pmin(1 - 1e-9,
      sum(wts * ifelse(pd == 1, 1 - eps, eps))))

    lk  <- vapply(parts, function(p) {
      pr <- .evaluate_rule_npe(p, x)
      if (pr == npe_cat_one[i]) 1 - eps else eps
    }, numeric(1))
    nw  <- wts * lk; tot <- sum(nw)
    wts <- if (tot > 1e-9) nw / tot else rep(1 / n_part, n_part)

    ess <- 1 / sum(wts^2)
    if (!is.nan(ess) && ess < n_part * RESAMPLE_THRESHOLD) {
      idx   <- sample.int(n_part, n_part, replace = TRUE, prob = wts)
      parts <- parts[idx]
      wts   <- rep(1 / n_part, n_part)
    }
  }
  rbinom(n_trials_npe, 1, probs)
}
```

```{r ch16_npe_train, eval=run_npe_track}
library(reticulate)
sbi   <- import("sbi")
torch <- import("torch")
utils <- import("sbi.utils")
infer <- import("sbi.inference")

n_train <- 20000L  # increase to 50k for production quality

train_npe_model <- function(simulator_fn, prior_low, prior_high, n_params,
                            model_name, n_sims = n_train) {
  cache_file <- sprintf("simdata/W16_npe_%s_train.rds", model_name)

  if (regenerate_fits || !file.exists(cache_file)) {
    cat("Generating", n_sims, "training simulations for", model_name, "...\n")
    set.seed(2026)
    thetas <- matrix(runif(n_sims * n_params,
                           min = prior_low, max = prior_high),
                     ncol = n_params, byrow = TRUE)
    responses <- future_map(seq_len(n_sims), function(i) {
      simulator_fn(thetas[i, ])
    }, .options = furrr_options(seed = TRUE), .progress = TRUE)
    y_mat <- do.call(rbind, responses)
    saveRDS(list(thetas = thetas, responses = y_mat), cache_file)
  } else {
    cached <- readRDS(cache_file)
    thetas <- cached$thetas
    y_mat  <- cached$responses
  }

  theta_t <- torch$tensor(thetas, dtype = torch$float32)
  x_t     <- torch$tensor(y_mat,  dtype = torch$float32)

  sbi_prior <- utils$BoxUniform(
    low  = torch$tensor(prior_low,  dtype = torch$float32),
    high = torch$tensor(prior_high, dtype = torch$float32)
  )

  cat("Training NPE for", model_name, "...\n")
  inference_obj <- infer$SNPE(
    prior = sbi_prior,
    density_estimator = utils$posterior_nn(
      model = "nsf", hidden_features = 64L, num_transforms = 5L,
      embedding_net = torch$nn$Sequential(
        torch$nn$Linear(as.integer(ncol(y_mat)), 128L),
        torch$nn$ReLU(),
        torch$nn$Linear(128L, 64L),
        torch$nn$ReLU(),
        torch$nn$Linear(64L, 32L)
      )
    )
  )
  inference_obj$append_simulations(theta_t, x_t)
  density_est <- inference_obj$train(
    training_batch_size = 256L, learning_rate = 5e-4,
    show_train_summary = TRUE
  )
  posterior <- inference_obj$build_posterior(density_est)
  list(posterior = posterior, thetas = thetas, responses = y_mat)
}
```

```{r ch16_boundaries, fig.height=5, fig.width=12}

  height   = seq(0.5, 4.5, length.out = 80),
  position = seq(0.5, 4.5, length.out = 80)
)

if (!is.null(fit_gcm)) {
  gcm_post <- fit_gcm$summary(variables = c("w", "c", "bias"))
  w_mean  <- gcm_post$mean[startsWith(gcm_post$variable, "w")]
  c_mean  <- gcm_post$mean[gcm_post$variable == "c"]
  b_mean  <- gcm_post$mean[gcm_post$variable == "bias"]

  obs_train <- as.matrix(common_data_df[, c("height", "position")])
  cat_train <- common_data_df$category_feedback

  gcm_grid_prob <- apply(as.matrix(grid_pts), 1, function(x_new) {
    sims <- vapply(seq_len(nrow(obs_train)), function(e) {
      d <- sum(w_mean * abs(x_new - obs_train[e, ]))
      exp(-c_mean * d)
    }, numeric(1))
    s1  <- sum(sims[cat_train == 1])
    s0  <- sum(sims[cat_train == 0])
    den <- s1 + s0
    if (den > 1e-9) b_mean * s1 / (b_mean * s1 + (1 - b_mean) * s0) else 0.5
  })

  r_mean <- fit_kalman$summary(variables = "r_value")$mean
  r_mat  <- diag(r_mean, 2)
  mu0    <- c(2.5, 2.5); mu1 <- c(2.5, 2.5)
  sig0   <- diag(10, 2);  sig1 <- diag(10, 2)

  kalman_update_fn <- function(mu, sigma, obs, r_mat) {
    S   <- sigma + r_mat; K <- sigma %*% solve(S)
    mu_new    <- as.numeric(mu + K %*% (obs - mu))
    IK        <- diag(2) - K
    sigma_new <- IK %*% sigma %*% t(IK) + K %*% r_mat %*% t(K)
    list(mu = mu_new, sigma = (sigma_new + t(sigma_new)) / 2)
  }

  for (i in seq_len(nrow(obs_train))) {
    x <- obs_train[i, ]
    if (cat_train[i] == 1) {
      upd  <- kalman_update_fn(mu1, sig1, x, r_mat); mu1 <- upd$mu; sig1 <- upd$sigma
    } else {
      upd  <- kalman_update_fn(mu0, sig0, x, r_mat); mu0 <- upd$mu; sig0 <- upd$sigma
    }
  }

  kalman_grid_prob <- apply(as.matrix(grid_pts), 1, function(x_new) {
    lp0 <- mvtnorm::dmvnorm(x_new, mean = mu0, sigma = sig0 + r_mat, log = TRUE)
    lp1 <- mvtnorm::dmvnorm(x_new, mean = mu1, sigma = sig1 + r_mat, log = TRUE)
    if (!all(is.finite(c(lp0, lp1)))) return(0.5)
    exp(lp1 - log(exp(lp0) + exp(lp1)))
  })

  eps_mean <- fit_rule$summary(variables = "error_prob")$mean
  rule_grid_prob <- apply(as.matrix(grid_pts), 1, function(x_new) {
    h <- x_new[1]; p <- x_new[2]
    preds <- c(
      as.integer(h > 2.5),
      as.integer(p < 2.5),
      as.integer(h > 2.0 & p > 3.0),
      as.integer(!(h < 2.0 | p < 2.0)),
      as.integer(h > 2.5 & p < 3.5)
    )
    mean(ifelse(preds == 1, 1 - eps_mean, eps_mean))
  })

  grid_pred <- grid_pts |>
    mutate(
      gcm_prob    = gcm_grid_prob,
      kalman_prob = kalman_grid_prob,
      rule_prob   = rule_grid_prob

      )
    )

  ggplot() +
    geom_tile(data = grid_pred,
              aes(x = position, y = height, fill = prob_cat1), alpha = 0.85) +
    stat_contour(data = grid_pred,
                 aes(x = position, y = height, z = prob_cat1),
```

## Conclusion: Two Tracks, Two Answers

Rather than viewing exemplar, prototype, and rule-based models as competing alternatives, we can appreciate the complementary insights they provide. This chapter introduced a **two-track comparison framework** that addresses a methodological problem:

**Track 1 (LOO/LFO-CV on Stan proxies)** provides a computationally convenient ranking based on out-of-sample predictive density, with the caveat that the rule model's score reflects the proxy's fit and the GCM/Kalman scores are approximate due to path-dependence. Track 1 is the right tool for quick screening.

**Track 2 (NPE on cognitive theories)** provides a methodologically fair ranking where all three models are fitted as forward simulators with no tractability advantage. Track 2 is the right tool for *cognitive* conclusions.

The two tracks agree on easy cases (the generating model wins in both) and may disagree on hard cases — where the disagreement is itself informative. The gap between the two tracks' rankings on rule-generated data makes the structural-bias cost visible and quantifiable.
